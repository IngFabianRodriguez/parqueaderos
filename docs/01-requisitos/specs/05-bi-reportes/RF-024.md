# RF-024 — Exportación de Datos a Excel/CSV y Conectores BI (Power BI / Google Data Studio)

## Metadata

- **Módulo**: BI y Reportes
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: report-service, export-service, api-gateway

---

## User Story

**Como** analista de negocio **quiero** exportar los datos de reportes a Excel y CSV, y conectar herramientas de BI como Power BI y Google Data Studio **para** realizar análisis avanzados y compartir dashboards con stakeholders.

---

## Objetivo

El sistema debe proporcionar mecanismos de exportación de datos de reportes en formatos estructurados (XLSX, CSV) y exponer endpoints API compatibles con herramientas de visualización BI (Power BI, Google Data Studio, Looker) mediante soporte para OData y JSON bidireccional.

---

## Actores

- **Actor principal**: Analista de negocio / Superadmin
- **Actores secundarios**: Operador (solo exportación propia), herramientas de BI (consumo programático)

---

## Precondiciones

- El usuario autenticado tiene rol superadmin o analista (con permisos de exportación).
- Los datos solicitados existen en el rango de fechas especificado.
- El volumen de datos a exportar no supera el límite configurado (ver Casos de Borde).

---

## Postcondiciones

### En caso de éxito

- El archivo exportado se genera y se entrega al usuario o se deja disponible para descarga.
- Se registra en auditoría el evento de exportación con tipo de formato y cantidad de registros.
- Para conectores BI: el endpoint responde con formato JSON/OData válido para el consumo de la herramienta.

### En caso de falla

- El sistema retorna mensaje de error con causa específica.
- No se entrega archivo parcial; si la exportación es > 10MB se hace async.
- El evento de falla se loguea con correlation id.

---

## Flujo Funcional

### Flujo Principal: Exportación Manual a Excel/CSV

1. El usuario genera un reporte (RF-023) y visualiza los resultados.
2. El usuario hace clic en el botón "Exportar" y selecciona formato (XLSX o CSV).
3. El sistema valida que el volumen de datos no exceda 50,000 registros para exportación sincrónica.
4. Si excede, el sistema ofrece exportar de forma asíncrona (background job).
5. El sistema genera el archivo aplicando los mismos filtros del reporte visualizado.
6. El sistema retorna archivo descargable con nombre: `{tipo_reporte}_{sede}_{fecha_inicio}_{fecha_fin}.{ext}`.
7. El sistema registra la exportación en auditoría.

### Flujo Alternativo: Exportación Asíncrona (> 50k registros)

1. El usuario selecciona formato y marca "Generar en segundo plano".
2. El sistema crea job encolado con prioridad normal.
3. El sistema retorna 202 Accepted con job_id y endpoint de status.
4. Cuando el job completa, el archivo queda disponible para descarga por 24h.
5. El usuario recibe notificación in-app cuando el archivo está listo.

### Flujo Alternativo: Connector Power BI (OData)

1. El analista configura en Power BI una nueva fuente OData.
2. Power BI se autentica con token JWT del sistema (grant type: client_credentials con scope limitado).
3. El sistema valida scopes y retorna metadata OData ($metadata).
4. Power BI consume los datasets expuestos como EntitySets (transactions, tickets, spaces).
5. El sistema filtra datos según los permisos del token (row-level security por sede).

### Flujo Alternativo: Connector Google Data Studio (JSON REST API)

1. El analista configura en Data Studio un conector "Generic JSON/Rest".
2. Data Studio solicita autenticación; el usuario ingresa credenciales OAuth2 del sistema.
3. El sistema valida y genera token limitado.
4. Data Studio consume los endpoints de reportes con filtros en query params.
5. Los datos se mapean a campos del Data Studio schema.

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Exportación > 50,000 registros sincrónica | Sistema rechaza con mensaje: "Exporte más de 50k registros de forma asíncrona" |
| Exportación async > 500,000 registros | Sistema rechaza con mensaje: "Volumen máximo de exportación: 500k registros" |
| Token OAuth2 expirado durante consumo BI | Sistema retorna 401 con header WWW-Authenticate; la herramienta debe re-autenticar |
|Rate limit en consumo BI (> 100 req/min) | Sistema retorna 429 con Retry-After; herramienta debe backoff |
| Formato CSV con caracteres especiales (tildes, ñ) | Encoding UTF-8 con BOM para compatibilidad Excel |
| Vacío en rango de fechas | Archivo generado con headers pero 0 filas; leyenda "Sin datos para el período seleccionado" |
| Permiso insuficiente en sede para token BI | Sistema filtra solo los datos autorizados (row-level security); no expone error 403 al tool |
| Columnas con nombres muy largos (> 31 chars Excel) | Truncar a 31 caracteres con hash de seguimiento para trazabilidad |

## Criterios de Aceptación

1. **CA-024-01**: Un usuario puede exportar un reporte de ingresos a XLSX y el archivo se abre correctamente en Microsoft Excel sin pérdida de datos.
2. **CA-024-02**: Un usuario puede exportar un reporte de ingresos a CSV y el archivo mantiene encoding UTF-8 y se abre correctamente en Google Sheets.
3. **CA-024-03**: Power BI puede conectarse vía OData y listar los EntitySets disponibles sin error de autenticación.
4. **CA-024-04**: Power BI aplica row-level security por sede según los scopes del token OAuth2.
5. **CA-024-05**: Google Data Studio puede consumir los endpoints JSON y mapear los campos al schema correctamente.
6. **CA-024-06**: Exportaciones de más de 50k registros se procesan de forma asíncrona y completan en menos de 5 minutos.
7. **CA-024-07**: Cada exportación genera un registro en auditoría con usuario, formato, cantidad de registros y hash de integridad.
8. **CA-024-08**: Archivos exportados incluyen suma de verificación (SHA256) para validación de integridad.

## Endpoints API Involucrados

- `POST /api/v1/exports` — Crear exportación sincrónica o encolar async
- `GET /api/v1/exports/{job_id}` — Consultar estado de job async
- `GET /api/v1/exports/{job_id}/download` — Descargar archivo generado
- `GET /api/v1/exports/catalog` — Listar templates de exportación predefinidos
- `GET /odata/v1/transactions` — Endpoint OData para Power BI
- `GET /odata/v1/tickets` — Endpoint OData para Power BI
- `GET /api/v1/bi/reports/{tipo}` — Endpoint JSON para Google Data Studio / herramientas REST
- `POST /api/v1/auth/token/BI` — Generar token OAuth2 con scopes específicos para BI

## Métricas de Éxito

- Tasa de exportaciones exitosas ≥ 99%
- Tiempo de generación de archivos ≤ 30s para < 10k registros
- Tiempo de generación async ≤ 5 min para < 500k registros
- Cantidad de conectores BI activos (Power BI + Data Studio) ≥ 1 por cliente
- Satisfacción del analista con herramientas de exportación ≥ 4/5

---

## Notas

- Se recomienda usar library para generación XLSX (Apache POI o similar en backend, o exceljs en Node.js) para no depender de servicios externos.
- La paginación OData usa $top y $skip; máximo page size = 1000 registros.
- Para tokens BI se usa OAuth2 con grant_type=client_credentials, scopes limitados a lectura, y TTL de 1 hora con refresh automático.
- Los archivos exportados de forma async se almacenan en almacenamiento local o cloud storage (S3-compatible) con lifecycle de 24h.
- El hash SHA256 se calcula sobre el archivo completo antes de comprimir y se incluye en el nombre del archivo o en metadata.
