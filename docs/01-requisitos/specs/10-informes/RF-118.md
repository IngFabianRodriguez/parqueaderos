# RF-118 — Comparativa Automática de Períodos

## Metadata

- **Módulo**: Informes
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: reporting-service, billing-service

---

## User Story

**Como** tenant_admin **quiero** ver la comparativa automática entre el período seleccionado y el período anterior **para** identificar tendencias de crecimiento o Decline en ingresos, ocupación y productividad.

---

## Objetivo

El sistema debe calcular y mostrar automáticamente, junto a cualquier reporte que Involucre un rango de fechas, la comparación con el período inmediatamente anterior de igual duración. Se debe mostrar la variación absoluta y el porcentaje de variación (%) con respecto al período anterior.

---

## Actores

- **tenant_admin**: Visualiza la comparativa
- **sede_admin**: Visualiza la comparativa de su sede
- **reporting-service**: Calcula los dos períodos y la variación

---

## Precondiciones

- RF-117 (filtrado de reportes) debe estar implementado
- Debe existir al menos un dato en el período seleccionado y/o en el período anterior
- Los períodos se calculan automáticamente: si el usuario selecciona del 1 al 31 de mayo, el período anterior es del 1 al 30 de abril

---

## Postcondiciones

### En caso de éxito

- La comparativa se muestra junto con el reporte principal
- Si no hay datos en uno de los períodos, se indica "sin datos" en lugar de un valor

### En caso de falla

- Si ambos períodos están vacíos, se muestra el reporte con variaciones nulas
- Si falla el cálculo de un período, se intenta con el otro y se marca la comparativa como parcial

---

## Flujo Funcional

1. El usuario ejecuta cualquier reporte con un rango de fechas (RF-117)
2. El `reporting-service` recibe `fecha_inicio` y `fecha_fin`
3. Calcula `periodo_actual`: desde `fecha_inicio` hasta `fecha_fin`
4. Calcula `periodo_anterior`: la misma duración en días inmediatamente antes
   - `diff_dias = fecha_fin - fecha_inicio`
   - `periodo_anterior_inicio = fecha_inicio - diff_dias`
   - `periodo_anterior_fin = fecha_inicio - 1 día`
5. Ejecuta la consulta del reporte para `periodo_actual`
6. Ejecuta la misma consulta para `periodo_anterior`
7. Calcula: `variacion_absoluta = valor_actual - valor_anterior`
8. Calcula: `variacion_porcentual = ((valor_actual - valor_anterior) / valor_anterior) * 100` (si valor_anterior ≠ 0)
9. Retorna ambos períodos con la comparativa

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| fecha_inicio | date | Inicio del período a comparar | Si |
| fecha_fin | date | Fin del período a comparar | Si |
| [filtros RF-117] | mixed | Cualquier otro filtro del RF-117 | No |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| periodo_actual | object | Datos del período seleccionado |
| periodo_anterior | object | Datos del período anterior de igual duración |
| comparativa | object | Objeto con variación absoluta y porcentual |
| comparativa.variacion_absoluta | decimal | Diferencia (actual - anterior) |
| comparativa.variacion_porcentual | decimal | Porcentaje de cambio (puede ser negativo) |
| comparativa.tendencia | enum | `up` (aumento), `down` (disminución), `stable` (< 1% cambio), `no_data` |
| dias_comparados | integer | Cantidad de días de cada período |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Período actual con datos, anterior sin datos | Muestra datos actuales; variación se marca como `no_data` |
| Período anterior con datos, actual sin datos | Muestra datos anteriores; variación se marca como `no_data` |
| Ambos períodos sin datos | Retorna vacío, variación = 0, tendencia = `no_data` |
| Período anterior = 0 (división por cero) | Variación porcentual se marca como `infinito` (mostrar "N/A") |
| Variación menor a 1% | Se clasifica como `stable` en lugar de `up` o `down` |
| Período seleccionado cruza año nuevo | El período anterior se calcula correctamente en años diferentes |

---

## Criterios de Aceptación

1. Si el usuario selecciona mayo (31 días), el período anterior debe ser los 31 días previos (30 de abril hacia atrás)
2. La `variacion_porcentual` se calcula con 2 decimales de precisión
3. La comparativa se muestra en todo reporte que use el filtro de fechas (RF-117)
4. Si `valor_anterior = 0` y `valor_actual > 0`, la variación porcentual se muestra como "+∞%" o "N/A"
5. La tendencia se clasifica automáticamente: `up` si > +1%, `down` si < -1%, `stable` en otro caso

## Endpoints API Involucrados

- `GET /api/v1/reports/income` — Con `include_comparison=true` retorna la comparativa
- `GET /api/v1/reports/occupancy` — Con `include_comparison=true` retorna la comparativa
- `GET /api/v1/reports/{report_type}?include_comparison=true` — Parámetro genérico

---

## Métricas de Éxito

- Tiempo adicional de respuesta por incluir comparativa (target: < 500ms)
- Precisión del cálculo de variación porcentual (target: 0 errores)
- Porcentaje de reportes ejecutados con comparativa habilitada (target: > 70%)

---

## Notas

- La comparativa es paramétrica: se activa con `include_comparison=true` en la query
- Para reportes que no Involucren valores numéricos (ej: reportes de estado), la comparativa puede no ser applicable y se omite
- El rango de fechas del período anterior se muestra en la UI para transparencia del usuario
- Se recomienda que la UI muestre flechas de color (verde ↑, rojo ↓) para facilitar lectura rápida
