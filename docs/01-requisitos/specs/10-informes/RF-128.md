# RF-128 — Exportación de Reportes (XLSX, CSV, PDF)

## Metadata

- **Módulo**: Informes
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: reports-service, export-service

---

## User Story

**Como** administrador de sede **quiero** exportar cualquier reporte a Excel, CSV o PDF **para** poder compartir la información con stakeholders, archivarla o analizarla en herramientas externas.

---

## Objetivo

El sistema debe permitir que cualquier reporte generado en el módulo de informes pueda ser exportado en formato Excel (.xlsx), CSV y PDF. La exportación debe mantener la estructura y datos del reporte tal como se visualiza en pantalla, incluyendo headers, totales y formateo.

---

## Actores

- **Actor principal**: Administrador de sede (`tenant_admin`)
- **Actor secundario**: Superadmin, Supervisor de operaciones, Administrador de empresa B2B (`fleet_admin`)

---

## Precondiciones

- El usuario tiene permisos de lectura sobre el reporte que desea exportar.
- El reporte ha sido generado y está disponible en la sesión actual.

---

## Postcondiciones

### En caso de éxito

- El archivo se genera y se descarga automáticamente o se presenta un link de descarga.
- El archivo contiene todos los datos del reporte con el formato correcto.

### En caso de falla

- Si el reporte está vacío, la exportación se genera con headers y mensaje "Sin datos para el período seleccionado".
- Si el formato solicitado no es soportado, se retorna error 400.
- Si el reporte excede el límite de filas para exportación (50,000), se retorna warning y se genera el archivo con los primeros 50,000 registros.

---

## Flujo Funcional

1. El usuario visualiza un reporte en el módulo de informes.
2. El usuario hace clic en el botón "Exportar" y selecciona el formato (XLSX, CSV o PDF).
3. El sistema valida que el usuario tenga permisos de exportación.
4. El sistema prepara la data del reporte:
   - **XLSX**: Genera workbook con sheets, aplica estilo a headers (negrita, color de fondo), columnas auto-ajustadas, formulas de suma para totales.
   - **CSV**: Genera archivo delimitado por comas, encoding UTF-8, sin fórmulas ni estilos.
   - **PDF**: Genera documento con layout de tabla, headers en cada página, pies de página con número de página y fecha de generación.
5. El sistema retorna el archivo para descarga o lo almacena temporalmente para link de descarga.

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| reporte_id | UUID | Identificador del reporte a exportar | Sí |
| formato | Enum | XLSX, CSV, PDF | Sí |
| incluir_graficos | Boolean | Incluir gráficos si el formato lo soporta (solo XLSX) | No (default: false) |
| pagina_tamanio | String | Tamaño de página para PDF: A4, LETTER, LEGAL | No (default: A4) |
| orientacion | Enum | PORTRAIT, LANDSCAPE | No (default: PORTRAIT) |
| columnas_seleccionadas | Array | Lista de columnas a incluir (si se omite, todas) | No |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| archivo_descarga | Binary | Archivo en el formato seleccionado |
| nombre_archivo | String | Nombre sugerido: `{reporte_nombre}_{YYYYMMDD_HHmmss}.{ext}` |
| tamano_bytes | Integer | Tamaño del archivo en bytes |
| tipo_contenido | String | MIME type: application/vnd.openxmlformats..., text/csv, application/pdf |
| filas_incluidas | Integer | Cantidad de filas incluidas en el archivo |
| url_temporal | String | URL temporal de descarga (validez 24h) si se almacenó |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Reporte vacío | Se genera archivo con headers y mensaje "Sin datos" |
| >50,000 filas | Se genera archivo con primeros 50,000 y se incluye nota "Datos truncados — máximo 50,000 filas" |
| Caracteres especiales (ñ, acentos) | Se codifican correctamente en todos los formatos (UTF-8) |
| Columnas muy anchas (texto largo) | Se truncan a 32767 caracteres en XLSX; en PDF se ajustan |
| Nombres de columna con caracteres inválidos para文件名 | Se sanitizan automáticamente (se reemplazan con underscore) |
| Exportación de reporte que expiró | Retorna error 400 "Reporte no disponible — por favor genere uno nuevo" |
| Formato CSV con valores que contienen comas | Se encierran en comillas dobles |
| PDF con más de 100 páginas | Se genera documento paginado; cada página tiene header y footer |

---

## Criterios de Aceptación

1. Cualquier reporte del módulo puede exportarse en los tres formatos: XLSX, CSV y PDF.
2. Los datos exportados coinciden exactamente con los datos visibles en pantalla en el momento de la exportación.
3. El archivo XLSX es abrible en Microsoft Excel sin errores de formato o fórmulas.
4. El archivo CSV es abrible en cualquier editor de texto o herramienta de hojas de cálculo.
5. El archivo PDF mantiene legibilidad y se puede imprimir correctamente en tamaño A4.
6. La exportación de un reporte de hasta 10,000 filas no toma más de 10 segundos.
7. Los archivos exportados incluyen timestamp de generación en el nombre.
8. Archivos temporal de descarga expiran después de 24 horas.

## Endpoints API Involucrados

- `POST /api/v1/reports/{reporte_id}/export` — Genera la exportación en el formato solicitado
- `GET /api/v1/reports/exports/{export_id}/download` — Descarga un archivo de exportación previamente generado

---

## Métricas de Éxito

- Exportación XLSX < 10s para reportes de hasta 10,000 filas.
- Exportación CSV < 5s para reportes de hasta 10,000 filas.
- Exportación PDF < 15s para reportes de hasta 5,000 filas.
- Cero errores de formato en archivos XLSX abiertos en Excel.

---

## Notas

- La exportación en PDF utiliza una biblioteca de rendering que soportael estándar PDF/A para archivo.
- Para reportes muy grandes, se recomienda usar CSV que es más rápido de generar.
- Los archivos exportados no se almacenan permanentemente en el servidor; solo se mantienen como descarga temporal (24h) o se envían directamente.
- Si el usuario tiene configuración regional diferente, los separadores de decimales y miles se ajustan al formato de la región del tenant.
