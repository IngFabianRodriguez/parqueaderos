# RF-155 — Configuración de Tarifas y Planes

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: tariff-service, site-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** configurar las tarifas de mis sedes **para** definir cómo se cobrará a los clientes por el uso del parqueadero.

---

## Objetivo

El sistema permite configurar tarifas por sede, con estructura flexible que soporta: tarifas por minuto/hora/día, franjas horarias diferentes, tarifas para tipos de espacio distintos, y tarifas especiales para clientes frecuentes o corporativos.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede

---

## Precondiciones

- La sede existe y está activa (RF-154).
- El admin tiene permisos de edición en la sede.

---

## Postcondiciones

### En caso de éxito

- Las tarifas quedan guardadas y aplicables a nuevos tickets.
- Se muestra confirmación de cambios.

### En caso de falla

- Las tarifas anteriores se mantienen sin cambios.
- Se muestra error si los datos son inconsistentes.

---

## Flujo Funcional

### Crear Tarifa Base

1. El admin abre la sede, pestaña "Tarifas".
2. Selecciona "Nueva Tarifa".
3. Define: nombre, tipo (MINUTO, HORA, DIA, MENSUAL), monto, duración en minutos/horas.
4. Selecciona a qué tipo(s) de espacio aplica.
5. Guarda.

### Crear Tarifa con Franjas Horarias

1. Admin crea nueva tarifa y activa "Franjas horarias".
2. Define franja: hora inicio, hora fin, monto.
3. Puede agregar múltiples franjas (ej: noche más barata).
4. El sistema aplica la franja correspondiente según hora de entrada.

### Crear Tarifa Especial

1. Admin selecciona "Tarifa Especial" (para clientes frecuentes o campañas).
2. Define: nombre, descuento (%), días aplicables, horario, vigencia.
3. Aplica a todos los espacios o tipos específicos.
4. Al guardar, la tarifa especial se activa automáticamente.

### Aplicar Tarifa a Cliente Corporativo

1. Admin accede a "Clientes" y abre un cliente.
2. En la pestaña "Tarifas especiales", selecciona la tarifa asignada.
3. El sistema aplica la tarifa corporativas a todos los tickets del cliente.

---

## Estructura de Tarifas

```
TARIFA HORA (ejemplo):
  - Primera hora: $3.000
  - Horas adicionales: $2.000/hora
  - Tarifa máxima día: $25.000
  - Franja noche (22:00-06:00): $5.000 fijo

TARIFA DIA:
  - Tarifa full day: $25.000
  - Tarifa noche (18:00-08:00): $15.000

TARIFA MENSUAL:
  - Plan básico: $250.000/mes
  - Plan VIP: $400.000/mes
```

---

## Datos de Entrada — Tarifa

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| nombre | String(50) | Nombre descriptivo | Si |
| tipo | Enum | MINUTO, HORA, DIA, MENSUAL | Si |
| monto_base | Decimal | Monto de la tarifa | Si |
| duracion_minutos | Integer | Duración (para MINUTO/HORA) | Si |
| monto_fraccion | Decimal | Monto por fracción adicional | No |
| tarifa_maxima | Decimal | Tope diario/mensual | No |
| hora_inicio_franaja | Time | Inicio de franja especial | No |
| hora_fin_franaja | Time | Fin de franja especial | No |
| monto_fracaja | Decimal | Monto de la franja | No |
| espacio_tipos | Array[UUID] | Tipos de espacio aplica | Si |
| activa | Boolean | Si está activa | Si |
| fecha_vigencia_inicio | Date | Desde cuándo aplica | Si |
| fecha_vigencia_fin | Date | Hasta cuándo aplica | No |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Conflicto de franjas | Error: "Las franjas no pueden overlap. Revise horas." |
| Tarifa sin vigencia fin | Se interpreta como "indefinida". |
| Tarifa para espacio inactivo | Warning: "Este tipo de espacio está inactivo." |
| Sin tarifa para espacio | Error: "Debe existir al menos una tarifa por tipo de espacio." |
| Cambiar tarifa con tickets abiertos | Los tickets abiertos mantienen la tarifa anterior. |

---

## Criterios de Aceptación

1. Las tarifas nuevas aplican inmediatamente a tickets que se creen después del cambio.
2. El historial de tarifas se mantiene para auditorías.
3. El admin puede ver un comparativo de tarifas actuales vs. anteriores.
4. Las franjas horarias se aplican correctamente según la hora de entrada del vehículo.
5. Las tarifas corporativas overridean las tarifas base del cliente.
6. El sistema permite configurar tarifas sin límite de complejidad.

## Endpoints API Involucrados

- `GET /api/v1/admin/sites/{site_id}/tariffs` — Listar tarifas
- `POST /api/v1/admin/sites/{site_id}/tariffs` — Crear tarifa
- `PUT /api/v1/admin/tariffs/{id}` — Actualizar tarifa
- `DELETE /api/v1/admin/tariffs/{id}` — Eliminar tarifa
- `GET /api/v1/admin/tariffs/history/{site_id}` — Historial de cambios

---

## Notas

- Las tarifas deben mantener historial para resolver disputas de cobros.
- Se recomienda nunca eliminar una tarifa, solo inactivarla.
