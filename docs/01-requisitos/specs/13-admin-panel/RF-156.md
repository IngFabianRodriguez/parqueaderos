# RF-156 — Gestión de Descuentos, Cupones y Promociones

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: discount-service, ticket-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** crear descuentos y cupones **para** ejecutar campañas comerciales y fidelizar clientes.

---

## Objetivo

El panel permite crear y gestionar descuentos y cupones que aplican a tickets de parqueadero. Se pueden definir por porcentaje o monto fijo, con límites de uso, vigencia, y condiciones específicas.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede

---

## Precondiciones

- El admin tiene permisos de configuración en la sede.
- No existe otro cupón con el mismo código.

---

## Postcondiciones

### En caso de éxito

- El cupón queda activo y disponible para aplicar.
- Se muestra código para compartir.

### En caso de falla

- No se crea el cupón.
- Se muestra error (código duplicado, datos inválidos).

---

## Flujo Funcional

### Crear Cupón de Descuento

1. Admin accede a "Descuentos y Cupones".
2. Hace clic en "Nuevo Cupón".
3. Define: código (único), tipo (PORCENTAJE/MONTO_FIJO), valor, máximo usos, vigencia.
4. Opcional: establece condiciones (mínimo de horas, tipos de espacio, clientes específicos).
5. Guarda.

### Aplicar Cupón a Ticket

1. Operador recibe el cupón del cliente (código).
2. En el ticket abierto, operador ingresa el código.
3. Sistema valida: existe, activo, no expirado, usos disponibles.
4. Sistema aplica el descuento al total y recalcula.
5. Ticket se marca con código usado.

---

## Datos de Entrada — Cupón

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| codigo | String(20) | Código único alfanumérico | Si |
| nombre | String(100) | Nombre de la campaña | Si |
| tipo_descuento | Enum | PORCENTAJE, MONTO_FIJO | Si |
| valor | Decimal | Porcentaje o monto | Si |
| max_usos | Integer | Usos máximos total | Si |
| max_usos_por_cliente | Integer | Usos por cliente | No |
| fecha_inicio | DateTime | Inicio de vigencia | Si |
| fecha_fin | DateTime | Fin de vigencia | No |
| min_horas | Integer | Mínimo de horas para aplicar | No |
| requiere_cliente | Boolean | Solo para clientes registrados | No |
| espacio_tipos | Array[UUID] | Tipos de espacio aplicables | No |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Código ya existe | Error: "Este código ya existe. Use otro." |
| Cupón expirado | Sistema rechaza: "Este cupón ha expirado." |
| Usos agotados | Sistema rechaza: "Este cupón ya no tiene usos disponibles." |
| Descuento mayor al ticket | Se aplica hasta cubrir el total (no genera saldo). |
| Cupón en ticket ya pagado | No permite aplicar; debe hacer reverso primero. |

---

## Criterios de Aceptación

1. Los cupones son únicos y case-insensitive.
2. Un cupón puede usarse una o múltiples veces según configuración.
3. Los cupones expirados no son aplicables.
4. El sistema registra cada uso con timestamp, cliente y ticket.
5. Un mismo cliente puede usar el mismo cupón según el límite por cliente.
6. El admin puede desactivar un cupón en cualquier momento.
