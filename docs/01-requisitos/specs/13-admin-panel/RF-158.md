# RF-158 — Gestión de Dispositivos IoT (Talanqueras y ANPR)

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: device-service, site-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** registrar y configurar los dispositivos IoT (talanqueras, cámaras ANPR) **para** integrar el hardware con el sistema.

---

## Objetivo

El sistema permite registrar dispositivos en cada sede, configurar sus parámetros de comunicación (IP, puerto), ver su estado en tiempo real, y diagnosticar problemas de conexión.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Técnico de soporte

---

## Flujo Funcional

### Registrar Dispositivo

1. Admin abre la sede, pestaña "Dispositivos".
2. Selecciona "Agregar Dispositivo".
3. Define: tipo (TALANQUERA/ANPR), nombre, IP, puerto, modelo.
4. Sistema prueba conexión.
5. Si exitoso, guarda y asocia a la sede.

### Ver Estado de Dispositivo

1. Admin abre la lista de dispositivos.
2. Ve columnas: Nombre, Tipo, Sede, Estado (ONLINE/OFFLINE/ERROR), Último heartbeat.
3. Estado se actualiza cada 30 segundos.

---

## Datos de Entrada — Dispositivo

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| nombre | String(50) | Nombre descriptivo | Si |
| tipo | Enum | TALANQUERA, ANPR | Si |
| modelo | String(50) | Modelo del dispositivo | No |
| ip | String(15) | Dirección IP | Si |
| puerto | Integer | Puerto de comunicación | Si |
| sede_id | UUID | Sede a asignar | Si |

---

## Criterios de Aceptación

1. Un dispositivo solo puede pertenecer a una sede.
2. El sistema marca OFFLINE si no hay heartbeat en 60 segundos.
3. El admin recibe alerta si un dispositivo pasa a estado ERROR.
4. Se puede reconfigurar IP/puerto sin eliminar el dispositivo.
