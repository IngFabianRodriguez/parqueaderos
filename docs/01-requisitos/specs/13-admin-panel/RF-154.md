# RF-154 — Configuración de Sedes

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: site-service, tenant-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** crear y configurar las sedes (parquesaderos) de mi cuenta **para** gestionar la infraestructura de cada ubicación.

---

## Objetivo

El panel permite crear sedes, configurar sus datos básicos (nombre, dirección, horarios), tipos de espacios disponibles, tarifas base, y asociar dispositivos (talanqueras, cámaras ANPR). Cada sede es una entidad independiente con su propia disponibilidad.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede

---

## Precondiciones

- El admin tiene sesión activa con rol ADMIN_TENANT o rol con permiso `sedes.crear`.
- El número de sedes no ha alcanzado el límite del plan (RF-153).

---

## Postcondiciones

### En caso de éxito

- La sede queda creada con configuración inicial.
- Se habilita la gestión de espacios, tarifas y dispositivos.
- Se registra en auditoría la creación de la sede.

### En caso de falla

- No se crea la sede.
- Se muestra error (límite alcanzado, datos inválidos).

---

## Flujo Funcional

### Crear Sede

1. El admin accede a "Sedes" y hace clic en "Nueva Sede".
2. El sistema muestra formulario en pasos:
   - **Paso 1 — Datos Básicos**: nombre, dirección, ciudad, coordenadas GPS, timezone.
   - **Paso 2 — Horarios**: horario de atención (días y horas), días festivos opcionales.
   - **Paso 3 — Espacios**: tipos de espacio disponibles (cubierta, descubierta, VIP, moto), cantidad por tipo.
   - **Paso 4 — Dispositivos**: asociar talanquera(s) y cámara(s) ANPR.
3. El admin completa cada paso y guarda.
4. El sistema valida datos y crea la sede con estado `ACTIVA`.
5. El sistema registra en auditoría: `SITE_CREATED`.

### Configurar Horarios

1. Admin abre la configuración de una sede.
2. Accede a pestaña "Horarios".
3. Define horario de atención por día de la semana.
4. Puede agregar días festivos con horario especial.
5. El sistema aplica estos horarios para validaciones de entrada/salida.

### Configurar Tipos de Espacio

1. Admin abre la sede, pestaña "Espacios".
2. Define tipos de espacio con: nombre, cantidad, tarifa base.
3. Puede activar/inactivar tipos de espacio sin eliminarlos.
4. Cambios aplican inmediatamente.

### Asociar Dispositivos

1. Admin abre la sede, pestaña "Dispositivos".
2. Ve lista de dispositivos disponibles (sin asignar).
3. Selecciona talanquera(s) y ANPR para asociar.
4. Define el rol de cada dispositivo: entrada, salida, entrada/salida.
5. El sistema valida que el dispositivo no esté en otra sede.

## Datos de Entrada — Crear Sede

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| nombre | String(100) | Nombre de la sede | Si |
| direccion | String(255) | Dirección completa | Si |
| ciudad | String(50) | Ciudad | Si |
| latitud | Decimal | Latitud GPS | Si |
| longitud | Decimal | Longitud GPS | Si |
| timezone | String(50) | Zona horaria (ej: America/Bogota) | Si |
| horario_general | JSON | Horarios por día {dia: {apertura, cierre}} | Si |

---

## Datos de Entrada — Tipo de Espacio

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| nombre | String(50) | Nombre (ej: Cubierta, Descubierta) | Si |
| cantidad | Integer | Número de espacios | Si |
| tarifa_hora | Decimal | Tarifa por hora base | Si |
| requiere_reserva | Boolean | Si requiere reserva obligatoria | No |

---

## Datos de Salida — Sede

| Campo | Tipo | Descripción |
|-------|------|-------------|
| sede_id | UUID | Identificador único |
| tenant_id | UUID | Tenant al que pertenece |
| nombre | String | Nombre |
| direccion | String | Dirección |
| ciudad | String | Ciudad |
| latitud | Decimal | Latitud |
| longitud | Decimal | Longitud |
| estado | Enum | ACTIVA, INACTIVA, SUSPENDIDA |
| espacios_total | Integer | Total de espacios |
| espacios_disponibles | Integer | Disponibles en tiempo real |
| num_talanqueras | Integer | Cantidad de talanquera(s) |
| num_anpr | Integer | Cantidad de cámaras ANPR |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Límite de sedes alcanzado | Error: "Ha alcanzado el límite de sedes de su plan (X). Actualice para agregar más." |
| Coordenadas GPS inválidas | Error: "Coordenadas no válidas. Verifique la dirección." |
| Horario cierre antes de apertura | Error: "La hora de cierre debe ser posterior a la apertura." |
| Dispositivo ya asignado a otra sede | Error: "Este dispositivo ya está asociado a otra sede." |
| Sede con transacciones activas | No se puede inactivar; error: "Hay transacciones activas. Cierre el día primero." |
| Horario vacío | Se permite 24 horas (siempre abierta). |

---

## Criterios de Aceptación

1. El admin puede crear una sede con todos sus espacios y dispositivos en menos de 2 minutos.
2. La disponibilidad de espacios se calcula en tiempo real (< 5 segundos).
3. Cada cambio en configuración de sede queda registrado en auditoría.
4. Los dispositivos asociados muestran su estado (online/offline) en la vista de sede.
5. Los horarios se aplican correctamente en validaciones de tickets.
6. El admin puede inactivar una sede sin eliminar datos históricos.
7. Los espacios pueden reconfigurarse sin afectar tickets abiertos.

## Endpoints API Involucrados

- `POST /api/v1/admin/tenants/{tenant_id}/sites` — Crear sede
- `GET /api/v1/admin/tenants/{tenant_id}/sites` — Listar sedes
- `GET /api/v1/admin/sites/{id}` — Detalle de sede
- `PUT /api/v1/admin/sites/{id}` — Actualizar sede
- `PUT /api/v1/admin/sites/{id}/spaces` — Configurar espacios
- `PUT /api/v1/admin/sites/{id}/devices` — Asociar dispositivos
- `PUT /api/v1/admin/sites/{id}/schedule` — Configurar horarios

---

## Métricas de Éxito

- Creación de sede: < 5 segundos.
- Consulta de disponibilidad: < 1 segundo.
- 0 sedes con límites inconsistentes.
- 100% de dispositivos asociados visibles en dashboard.

---

## Notas

- Los horarios definidos aquí pueden reemplazarse temporalmente por días festivos (festivos nacionales no se configuran manualmente, se importan de fuente externa).
- La cantidad de espacios no puede ser 0; mínimo 1.
- Los dispositivos deben estar previamente registrados en el sistema (RF-109+).
