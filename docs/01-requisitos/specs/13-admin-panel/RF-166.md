# RF-166 — Programación de Tareas y Jobs

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Baja
- **Tipo**: Funcional
- **Servicios afectados**: scheduler-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** programar tareas automáticas **para** delegar trabajos repetitivos al sistema.

---

## Objetivo

El panel permite crear tareas programadas: envío de reportes periódicos, notificaciones de cumpleaños/vencimientos, y mantenimiento automático de datos.

---

## Actores

- **Actor principal**: Admin de tenant

---

## Flujo Funcional

### Crear Tarea Programada

1. Admin accede a "Herramientas > Tareas Programadas".
2. Selecciona "Nueva Tarea".
3. Define: nombre, tipo, frecuencia (diaria/semanal/mensual), hora, parámetros.
4. Sistema agenda la tarea.
5. Admin recibe notificación cuando se ejecuta.

---

## Tipos de Tareas Disponibles

| Tipo | Descripción |
|------|-------------|
| ENVIO_REPORTE | Envía reporte por email según programación |
| NOTIFICACION_MORA | Envía recordatorio a clientes con deuda |
| LIMPIEZA_DATOS | Elimina datos temporales o vencidos |
| SYNC_EXTERNO | Sincroniza datos con sistema externo |

---

## Criterios de Aceptación

1. Las tareas se ejecutan aunque el admin no esté logueado.
2. El sistema registra resultado de cada ejecución (éxito/falla).
3. El admin recibe alerta si una tarea falla 3 veces consecutivas.
4. Se puede pausar/reanudar tareas sin eliminarlas.

---

## Notas

- El scheduler interno no es para tareas de alta frecuencia; esas van en el motor de workers.
