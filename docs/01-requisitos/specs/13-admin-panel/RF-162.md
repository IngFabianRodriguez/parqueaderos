# RF-162 — Gestión de Wallis Virtuales (Billetera)

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: wallet-service, customer-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** ver y gestionar las billeteras virtuales de mis clientes **para** garantizar el correcto funcionamiento del prepago.

---

## Objetivo

El panel permite consultar el saldo de las billeteras, agregar fondos manualmente, revertir transacciones, y ver el historial completo de movimientos de cada billetera.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede

---

## Flujo Funcional

### Consultar Billetera

1. Admin accede a "Billeteras" en el menú.
2. Busca por cliente o vehículo.
3. Sistema muestra: saldo actual, último cargo, último descuento, estado.

### Agregar Fondos Manualmente

1. Admin abre la billetera del cliente.
2. Selecciona "Agregar Fondos".
3. Ingresa monto y motivo.
4. Confirma.
5. Sistema incrementa el saldo y registra en historial.

### Revertir Transacción

1. Admin abre historial de la billetera.
2. Identifica la transacción a revertir.
3. Selecciona "Revertir".
4. Sistema crea transacción negativa y ajusta el saldo.

---

## Datos de Salida — Billetera

| Campo | Tipo | Descripción |
|-------|------|-------------|
| wallet_id | UUID | Identificador único |
| cliente_id | UUID | Cliente asociado |
| saldo | Decimal | Saldo actual |
| fecha_ultimo_cargo | DateTime | Último incremento |
| fecha_ultimo_descuento | DateTime | Último decremento |
| estado | Enum | ACTIVA, BLOQUEADA, CANCELADA |
| total_cargado_mes | Decimal | Total cargado en el mes |
| total_gastado_mes | Decimal | Total gastado en el mes |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Saldo insuficiente | No se permite descuento; ticket debe pagarse por otro medio. |
| Billetera bloqueada | No permite cargos ni descuentos. |
| Revertir transacción ya revertida | Error: "Esta transacción ya fue revertida." |

---

## Criterios de Aceptación

1. El saldo se actualiza en tiempo real tras cada operación.
2. Toda operación de billetera genera una entrada en el historial.
3. Las revertir crea una transacción negativa que compensa la original.
4. El admin puede bloquear una billetera si detecta fraude.
