# RF-159 — Dashboard y Métricas del Admin Panel

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: analytics-service, dashboard-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** ver un dashboard con las métricas más importantes de mi operación **para** tomar decisiones informadas.

---

## Objetivo

El panel de administración muestra métricas clave en tiempo real: espacios disponibles, tickets abiertos, ingresos del día, vehículos bloqueados, y alertas. Los datos se actualizan automáticamente sin necesidad de recargar.

---

## Actores

- **Actor principal**: Admin de tenant, Superadmin
- **Actor secundario**: Supervisor de sede

---

## Flujo Funcional

### Ver Dashboard

1. Admin inicia sesión en el panel.
2. Sistema carga dashboard con widgets de métricas.
3. Admin ve datos actualizados automáticamente cada 30 segundos.
4. Admin puede hacer drill-down en cualquier métrica.

---

## Widgets del Dashboard

| Widget | Descripción | Actualización |
|--------|-------------|---------------|
| Espacios Disponibles | Total y porcentaje por sede | 30s |
| Tickets Abiertos | Cantidad y antigüedad promedio | 30s |
| Ingresos del Día | Total facturado vs. objetivo | 5min |
| Vehículos Bloqueados | Total y lista reciente | 1min |
| Alertas | Dispositivos offline, anomalías | Real-time |
| Tendencia Semanal | Gráfico de ocupación semanal | 1h |

---

## Criterios de Aceptación

1. El dashboard carga en < 3 segundos.
2. Las métricas se actualizan sin recargar la página.
3. Admin puede filtrar por sede.
4. Superadmin ve métricas de todos los tenants.
5. Se puede exportar el resumen en PDF.

---

## Notas

- Las métricas históricas se acceden desde el módulo de Informes (RF-128+).
