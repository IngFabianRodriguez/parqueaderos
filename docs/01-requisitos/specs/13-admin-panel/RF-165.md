# RF-165 — Configuración de Integraciones (API Keys)

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: integration-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** gestionar las API keys y webhooks de mi cuenta **para** integrar el sistema con aplicaciones de terceros.

---

## Objetivo

El panel permite crear, gestionar y revocar API keys para integración, así como configurar webhooks que recibirán eventos del sistema.

---

## Actores

- **Actor principal**: Admin de tenant

---

## Flujo Funcional

### Crear API Key

1. Admin accede a "Integraciones > API Keys".
2. Selecciona "Nueva API Key".
3. Ingresa nombre, permisos (scopes), fecha de expiración.
4. Sistema genera la clave y la muestra una sola vez.
5. Admin copia y guarda de forma segura.

### Configurar Webhook

1. Admin accede a "Integraciones > Webhooks".
2. Selecciona "Nuevo Webhook".
3. Define: URL, eventos a escuchar (ticket.created, payment.completed, etc.).
4. Sistema prueba la conexión con un ping.
5. Si exitoso, activa el webhook.

---

## Datos de Entrada — API Key

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| nombre | String(50) | Nombre descriptivo | Si |
| scopes | Array[String] | Permisos (read, write, admin) | Si |
| fecha_expiracion | DateTime | Cuándo expira (null = sin expiración) | No |
| ip_whitelist | Array[String] | IPs permitidas (opcional) | No |

---

## Datos de Entrada — Webhook

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| nombre | String(50) | Nombre descriptivo | Si |
| url | String(255) | Endpoint que recibe eventos | Si |
| eventos | Array[String] | Lista de eventos a recibir | Si |
| secreto | String | Para verificar payloads (auto-generado) | Si |
| activo | Boolean | Si está activo | Si |

---

## Criterios de Aceptación

1. Las API keys se muestran una sola vez al crear; después solo se puede ver el nombre y fecha.
2. Se puede revocar una key sin eliminarla (soft revoke).
3. Los webhooks incluyen retry automático si el endpoint no responde (3 intentos con backoff).
4. Cada webhook tiene log de últimos 100 intentos con estado.

---

## Notas

- Los scopes disponibles: `tickets.read`, `tickets.write`, `payments.read`, `customers.read`, `reports.read`.
- La secret del webhook se usa para calcular HMAC-SHA256 del payload.
