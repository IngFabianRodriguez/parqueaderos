# RF-161 — Gestión de Reportes y Vistas Guardadas

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: report-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** guardar vistas personalizadas de reportes **para** acceder rápidamente a la información que necesito sin reconfigurar filtros cada vez.

---

## Objetivo

El sistema permite guardar la configuración de cualquier reporte (filtros, columnas, orden) como una vista con nombre. Las vistas guardadas aparecen en el menú del admin para acceso rápido.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede

---

## Flujo Funcional

### Guardar Vista de Reporte

1. Admin configura un reporte con los filtros y columnas deseados.
2. Hace clic en "Guardar Vista".
3. Ingresa nombre y descripción opcional.
4. Sistema guarda la configuración.
5. La vista aparece en el menú "Mis Reportes".

### Cargar Vista Guardada

1. Admin accede a "Mis Reportes" en el menú.
2. Selecciona una vista guardada.
3. Sistema carga el reporte con todos los filtros aplicados.

---

## Criterios de Aceptación

1. Cada usuario ve solo sus propias vistas guardadas.
2. Las vistas guardadas incluyen: filtros, columnas visibles, ordenamiento.
3. Admin puede eliminar o renombrar sus vistas.
4. Las vistas son específicas por módulo de reporte.

---

## Notas

- Las vistas guardadas no son compartibles entre usuarios (funcionalidad futura).
