# RF-168 — Panel de Control del Superadmin (Vista Multi-Tenant)

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: superadmin-service, tenant-service, admin-service

---

## User Story

**Como** superadmin del SaaS **quiero** ver y gestionar todos los tenants desde un solo panel **para** tener visibilidad global y tomar decisiones operativas.

---

## Objetivo

El superadmin tiene un panel dedicado que muestra métricas consolidadas de todos los tenants, permite gestionar cuentas, ver uso de recursos, y detectar anomalías.

---

## Actores

- **Actor principal**: Superadmin del SaaS

---

## Flujo Funcional

### Ver Dashboard Global

1. Superadmin inicia sesión.
2. Ve dashboard con:
   - Total de tenants activos.
   - Ingresos consolidados del mes.
   - Uso de recursos (API calls, storage, bandwidth).
   - Alertas de riesgo (tenants próximos a expirar, uso excesivo).
3. Puede drill-down a cualquier tenant.

### Gestionar Tenant desde Vista Global

1. Superadmin abre la lista de tenants.
2. Selecciona un tenant.
3. Ve métricas específicas del tenant.
4. Puede realizar cualquier acción de gestión (suspender, cambiar plan, etc.).

---

## Widgets del Dashboard Superadmin

| Widget | Descripción |
|--------|-------------|
| Tenants Activos | Total y nuevos del mes |
| Ingresos Consolidados | Facturación total del mes |
| Top 10 Tenants | Por uso/ingresos |
| Alertas | Tenants en riesgo |
| Uso de Recursos | API calls, storage global |
| Tenants Trial | Próximos a vencer |

---

## Criterios de Aceptación

1. El dashboard carga en < 5 segundos.
2. Los datos se actualizan cada 15 minutos (no real-time).
3. Superadmin puede exportar la lista completa de tenants a CSV.
4. Las métricas de un tenant específico son comparables con el promedio.
5. El acceso a este panel requiere rol SUPERADMIN explícitamente.
