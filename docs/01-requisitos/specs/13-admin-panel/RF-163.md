# RF-163 — Gestión de Morosos y Bloqueos

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: collection-service, vehicle-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** gestionar la cartera de morosos y los bloqueos de vehículos **para** proteger la operación y garantizar el cobro.

---

## Objetivo

El panel muestra la lista de clientes con deuda pendiente, permite bloquear vehículos asociados, desbloquear cuando pagan, y ver el historial completo de la cartera de morosidad.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede, Operador

---

## Flujo Funcional

### Ver Cartera de Morosos

1. Admin accede a "Morosos" en el menú.
2. Ve lista: cliente, vehículo(s), monto pendiente, días en mora, último pago.
3. Puede filtrar por rango de días en mora, monto mínimo.

### Bloquear Vehículo

1. Admin selecciona un cliente moroso.
2. Selecciona el vehículo a bloquear.
3. Hace clic en "Bloquear".
4. Sistema registra bloqueo y envía notificación al cliente.
5. Si el vehículo intenta salir, la talanquera no abre (RF-141).

### Desbloquear Vehículo

1. Admin selecciona un cliente con bloqueo activo.
2. Verifica que el pago esté confirmado.
3. Hace clic en "Desbloquear".
4. Sistema libera el vehículo y limpia el estado de mora temporal.
5. Si hay otra deuda pendiente, permanece bloqueado con otro vehículo.

### Aplicar Pago y Desbloquear

1. Admin registra el pago (RF-135).
2. Sistema detecta que la deuda se cubre.
3. Sistema automáticamente desbloquea el vehículo.
4. Sistema registra en auditoría: `DEBT_CLEARED`.

## Datos de Salida — Moroso

| Campo | Tipo | Descripción |
|-------|------|-------------|
| cliente_id | UUID | Identificador del cliente |
| nombre | String | Nombre completo |
| telefono | String | Teléfono |
| vehiculos | Array | Vehículos con bloqueo |
| total_deuda | Decimal | Suma de todas las deudas pendientes |
| dias_mora | Integer | Días desde la primera deuda no pagada |
| ultimo_pago | DateTime | Fecha del último pago realizado |
| ultimo_pago_monto | Decimal | Monto del último pago |

---

## Estados del Vehículo

| Estado | Descripción |
|--------|-------------|
| ACTIVO | Sin bloqueo |
| BLOQUEADO_TEMPORAL | Bloqueado por mora (auto) |
| BLOQUEADO_MANUAL | Bloqueado por administrador |
| LIBERADO | Deuda pagada, desbloqueado |

---

## Criterios de Aceptación

1. El sistema bloquea automáticamente cuando un ticket supera X días pendiente (configurable).
2. El desbloqueo manual requiere justificación registrada en auditoría.
3. Un cliente puede tener múltiples vehículos; el bloqueo es por vehículo.
4. El desbloqueo de un vehículo no libera a otros vehículos bloqueados del mismo cliente.
5. Se envía notificación push/SMS al cliente al bloquear y al desbloquear.
