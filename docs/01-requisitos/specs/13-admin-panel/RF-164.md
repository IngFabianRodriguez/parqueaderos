# RF-164 — Exportación Masiva de Datos

## Metadata

- **Módulo**: 13 — Admin Panel
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: export-service, admin-service

---

## User Story

**Como** admin de tenant **quiero** exportar datos masivamente en diferentes formatos **para** usar la información en sistemas externos o para auditorías.

---

## Objetivo

El panel permite exportar datos de cualquier módulo (usuarios, tickets, pagos, clientes) en formatos estándar: Excel (.xlsx), CSV, y PDF. Las exportaciones pueden ser completas o filtradas.

---

## Actores

- **Actor principal**: Admin de tenant
- **Actor secundario**: Supervisor de sede

---

## Flujo Funcional

### Exportar Datos

1. Admin accede al módulo que desea exportar (ej: "Clientes").
2. Aplica los filtros deseados.
3. Hace clic en "Exportar".
4. Selecciona formato: XLSX, CSV, PDF.
5. Sistema genera el archivo y ofrece descarga.
6. Para exports grandes (> 10.000 filas), sistema envía email con link de descarga.

---

## Datos de Entrada

| Campo | Tipo | Descripción |
|-------|------|-------------|
| modulo | String | Módulo a exportar |
| filtros | JSON | Filtros aplicados |
| formato | Enum | XLSX, CSV, PDF |
| columnas | Array[String] | Columnas a incluir (opcional, default todas) |

---

## Criterios de Aceptación

1. El archivo se genera en < 30 segundos para hasta 10.000 registros.
2. Para exports grandes, el admin recibe email con link válido 24 horas.
3. Los exports incluyen un resumen con totales y fecha de generación.
4. Los permisos de exportación respetan RBAC (solo datos permitidos para el rol).

---

## Dependencias

**RFs relacionados**:

## Notas

- Las exportaciones se registran en el log de auditoría.
- Los archivos exportados incluyen máscara de datos sensibles (email parcial, teléfono parcial) según RGPD.
