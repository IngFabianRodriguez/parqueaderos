# RF-007 — Manejar placas ilegibles con registro manual

## Metadata

- **Módulo**: ANPR
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: anpr-service, parking-service

---

## User Story

**Como** Operador de sede **quiero** registrar manualmente la placa cuando el sistema ANPR no puede leerla **para** permitir la entrada y salida de vehículos cuando el OCR falla, sin bloquear la operación.

---

## Objetivo

Cuando el sistema ANPR reporta una confianza de OCR inferior al umbral mínimo (70%) o cuando el operador detecta que la placa leída es incorrecta, el operador puede ingresar manualmente la placa. Este registro manual genera un registro equivalente a uno automático, con la diferencia de que el campo 'metodo_registro' se marca como 'manual' y se incluye el ID del operador que realizó el registro.

---

## Actores

- Operador de sede
- [Actores secundarios]

---

## Precondiciones

      - Vehículo está presente en la entrada o salida
      - El operador tiene acceso a la app móvil o al panel de operaciones

---

## Postcondiciones

### En caso de éxito

      - Se crea registro de entrada o salida con método='manual' y operador_id
      - El sistema permite continuar el flujo normal (asignar espacio o calcular tarifa)

### En caso de falla

      - Vehículo ya tiene entrada sin cerrar con otra placa → se alerta al operador
      - Operador sin permisos de registro manual → se deniega el acceso

---

## Flujo Funcional

1. Sistema ANPR reporta placa con confianza < 70% o indica 'no_detectada'
1. Sistema muestra alerta en dashboard del operador con imagen de la placa
1. Operador verifica visualmente la placa y la ingresa manualmente en la app
1. Sistema recibe POST /api/v1/entradas/manual con placa, sede_id, operador_id
1. Sistema valida que no exista entrada activa duplicada para el mismo vehículo
1. Sistema crea registro de entrada con metodo_registro='manual' y operador_id
1. Sistema asigna espacio libre o, en salida, calcula tarifa normalmente
1. Talanquera se abre normalmente

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| placa | VARCHAR(20) | Placa ingresada manualmente por el operador | Sí |
| sede_id | UUID | Identificador de la sede | Sí |
| tipo_registro | VARCHAR | 'entrada' o 'salida' | Sí |
| operador_id | UUID | ID del operador que realiza el registro manual | Sí |
| imagen_evidencia | BINARY | Captura de la placa para auditoría | No |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| registro_id | UUID | — |
| placa | VARCHAR | — |
| metodo_registro | VARCHAR | manual |
| operador_id | UUID | — |
| timestamp | TIMESTAMP | — |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Placa ingresada no tiene formato válido | Se valida formato según normativa colombiana y se muestra error si no coincide |
| Entrada manual duplicada (mismo vehículo ya tiene entrada activa) | Se alerta al operador y se le pregunta si desea corregir o forzar |
| Operador intenta registro manual sin conexión | Se almacena en cola local y se sincroniza cuando haya conexión |

---

## Criterios de Aceptación

- Registro manual es rastreable hasta el operador específico que lo creó
- Todos los registros manuales incluyen evidencia fotográfica
- El sistema no permite más del 10% de registros manuales sobre automáticos (para detectar problemas de ANPR)

## Endpoints API Involucrados

- `POST /api/v1/entradas/manual — Registro manual de entrada`
- `POST /api/v1/salidas/manual — Registro manual de salida`

---

## Métricas de Éxito

- Tasa de registro manual < 10% del total de ingresos
- Rastreabilidad: 100% de registros manuales tienen operador_id
