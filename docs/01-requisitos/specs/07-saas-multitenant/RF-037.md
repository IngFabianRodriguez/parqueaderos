# RF-037 — Selección de Plan de Suscripción

## Metadata

- **Módulo**: SaaS Multi-Tenant — Suscripciones y Facturación
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: billing-service, tenant-service, plan-service

---

## User Story

**Como** administrador de un tenant **quiero** ver y seleccionar un plan de suscripción adecuado a mis necesidades **para** acceder a las funcionalidades que mi negocio requiere.

---

## Objetivo

El sistema debe presentar los planes de suscripción disponibles (Starter, Professional, Enterprise, Custom) con sus características, precios y límites, permitiendo al tenant_admin seleccionar y contratar el plan que mejor se ajuste a sus necesidades.

---

## Actores

- **Actor principal**: Tenant admin
- **Actor secundario**: Sistema ( Stripe checkout), Superadmin (para planes Custom)

---

## Precondiciones

- El tenant tiene estado `trial`, `active` (upgrade), `canceled` (reactivación) o `suspended` (reactivación).
- El tenant_admin tiene sesión activa.

---

## Postcondiciones

### En caso de éxito

- El tenant conoce los detalles, precios y límites de cada plan.
- El tenant selecciona un plan y es redirigido al flujo de pago.
- El plan seleccionado se activa una vez confirmado el pago.

### En caso de falla

- Si el pago falla, el tenant permanece en su estado actual sin cambio de plan.
- Si el plan Custom requiere aprobación, el tenant recibe instrucciones de contacto.

---

## Flujo Funcional

1. Tenant admin accede a Settings > Suscripción > "Cambiar plan".
2. Sistema muestra la página de selección de planes con:
   - Comparativa visual de planes: Starter, Professional, Enterprise, Custom.
   - Lista de features incluidas en cada plan (RF-056).
   - Límites: número de sedes, usuarios, transacciones mensuales, API calls.
   - Precio mensual y anual (con descuento del 20% para annual).
3. Tenant compara planes y hace clic en "Suscribirse" en el plan elegido.
4. Sistema redirige al Stripe Checkout (planes Starter, Professional, Enterprise) o muestra instructions para Custom.
5. Para planes Starter/Professional/Enterprise: Stripe Checkout muestra el precio y términos; el usuario ingresa datos de pago.
6. Al confirmar el pago, Stripe envía webhook `payment_succeeded`.
7. Sistema actualiza la suscripción del tenant con el nuevo plan.
8. Sistema genera evento `plan_upgraded` o `plan_changed`.
9. Sistema envía email de confirmación al tenant_admin.
10. Sistema actualiza los feature flags del tenant según el nuevo plan (RF-056).

### Planes Custom

1. Para plan Custom: el sistema muestra formulario de contacto (nombre empresa, número de vehículos, necesidades específicas).
2. El request se guarda como `custom_plan_requests` para revisión por el equipo de ventas.
3. El equipo de ventas contacta al tenant para definir términos y precio.
4. Una vez acordado, el superadmin crea manualmente la suscripción Custom desde el admin panel.

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| tenant_id | UUID | Identificador del tenant | Sí |
| plan_selected | enum | Plan elegido (starter, professional, enterprise, custom) | Sí |
| billing_cycle | enum | Ciclo de facturación (monthly, annual) | Sí |
| custom_plan_details | JSON | Detalles de necesidades para plan Custom | Sí (Custom) |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| plan | string | Plan activo del tenant |
| price | decimal | Precio mensual del plan |
| billing_cycle | string | Ciclo de facturación |
| features | array | Lista de features activas para el plan |
| limits | object | Límites del plan (sedes, usuarios, transacciones) |
| next_billing_date | datetime | Fecha del próximo cobro |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Upgrade desde trial | El plan se activa inmediatamente; se cobra el precio completo del ciclo. |
| Upgrade desde plan Starter a Professional | Prorrateo el día del upgrade; se cobra la diferencia del mes en curso. |
| Downgrade | Se programa para el final del ciclo actual (RF-041). |
| Pago falla durante checkout | Tenant permanece en plan actual; se muestra mensaje de error de Stripe. |
| Plan Custom solicitado | Se guarda request; el tenant ve "Nos pondremos en contacto en 24h". |

---

## Criterios de Aceptación

1. La página de selección de planes muestra todos los planes disponibles con sus precios y features.
2. El tenant puede seleccionar monthly o annual; el precio annual muestra un descuento visible del 20%.
3. El flujo de checkout con Stripe se completa sin errores en > 99% de los casos.
4. Al seleccionar un plan, el tenant ve inmediatamente los feature flags actualizados en su dashboard.
5. Los límites del plan (sedes, usuarios) se aplican al momento de la activación del plan.
6. Si el tenant intenta usar más sedes de las que su plan permite, recibe un error 403 con upgrade prompt.
7. El plan Custom requiere validación por parte del equipo de ventas; no se activa automáticamente.

## Endpoints API Involucrados

- `GET /api/v1/plans` — Listar planes disponibles
- `GET /api/v1/plans/{plan_id}` — Detalle de un plan
- `POST /api/v1/subscriptions/create-checkout` — Crear Stripe Checkout session
- `GET /api/v1/tenants/{tenant_id}/subscription` — Consultar suscripción actual

---

## Métricas de Éxito

- Conversion rate de visita a checkout > 60% para tenants en trial
- Mix de planes: Starter 40%, Professional 35%, Enterprise 20%, Custom 5% (benchmark)
- ARPU (Average Revenue Per User) por plan diferenciable
- Tiempo promedio del checkout < 3 minutos

---

## Notas

- Los precios de los planes se almacenan en la tabla `plans` y se muestran en la UI en la moneda local del tenant (COP para Colombia).
- El descuento por annual se calcula como 12 meses pagados al precio de 10 ( aproximación práctica al 16.7% de descuento real).
- Los feature flags de cada plan se definenen en RF-056 y se consultan cada vez que el usuario accede a una funcionalidad restringida.
