# RF-075 — Dashboard BI con Tendencias en Tiempo Real (Enterprise+)

## Metadata

- **Módulo**: SaaS Multi-Tenant — BI Avanzado (solo Enterprise+)
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: analytics-service, bi-service, site-service

---

## User Story

**Como** administrador de un tenant Enterprise **quiero** ver un dashboard con tendencias de ocupación, ingresos y transacciones en tiempo real **para** tomar decisiones operativas basadas en datos actualizados al minuto.

---

## Objetivo

El sistema debe exponer un dashboard de Business Intelligence que muestre métricas clave del negocio de estacionamiento en tiempo real: ocupación actual y trends, ingresos del día/mes, volumen de transacciones. Los datos se actualizan automáticamente cada 60 segundos sin necesidad de recargar la página. El dashboard está disponible únicamente para tenants Enterprise+.

---

## Actores

- **Actor principal**: Administrador del tenant (`tenant_admin`)
- **Actor secundario**: Usuario con rol `bi_viewer` o superior

---

## Precondiciones

- El tenant tiene plan Enterprise o superior.
- El actor tiene rol con permisos de visualización de BI.

---

## Postcondiciones

### En caso de éxito

- El dashboard carga con datos actuales de ocupación, ingresos y transacciones.
- Los gráficos se actualizan automáticamente cada 60 segundos.
- El actor puede filtrar por rango de fechas, sede, zona.

### En caso de falla

- Si el plan del tenant no es Enterprise+: mensaje de upgrade requerido.
- Si no hay datos disponibles: mensaje indicando "Sin datos disponibles para el período seleccionado".

---

## Flujo Funcional

1. El usuario accede a `GET /dashboard/bi` desde el admin panel.
2. El sistema verifica:
   - El plan del tenant es Enterprise+.
   - El usuario tiene rol con acceso a BI (`tenant_admin`, `bi_viewer`, `superadmin`).
   - Si alguna verificación falla, retorna página de error con mensaje apropiado.
3. El sistema consulta los datos para el período por defecto (hoy):
   - **Ocupación en tiempo real**: última lectura de cada espacio en cada sede.
   - **Ingresos del día**: suma de `amount` en `transactions` del día actual por sede.
   - **Transacciones del día**: conteo de `transactions` del día actual.
4. Se renderiza el dashboard con los datos cargados.
5. Cada 60 segundos, el frontend hace `GET /api/v1/analytics/realtime-summary` y actualiza los widgets sin recargar la página.

### Widgets del Dashboard

| Widget | Métrica | Descripción |
|--------|---------|-------------|
| Ocupación en tiempo real | % de espacios ocupados vs total | Gauge o círculo con porcentaje |
| Tendencia de ocupación (24h) | % ocupación por hora | Gráfico de línea |
| Ingresos del día | Suma de dinero | Tarjeta con monto y variación vs ayer |
| Ingresos por sede | Distribución | Gráfico de barras horizontal |
| Transacciones por hora | Conteo | Gráfico de línea |
| Tiempo promedio de permanencia | Minutos | Tarjeta con promedio |
| Ingresos del mes | Suma acumulada | Tarjeta con progreso vs meta |
| Top 5 espacios con mayor rotación | Espacios | Lista ordenada |

### Filtros disponibles

- **Rango de fechas**: Hoy, Ayer, últimos 7 días, último mes, mes actual, personalizado.
- **Sede**: Todas o específica.
- **Zona**: Todas o específica (solo si hay filtro de sede).
- **Granularidad**: Hora, Día, Semana, Mes.

## Datos de Salida

### Resumen en tiempo real

| Campo | Tipo | Descripción |
|-------|------|-------------|
| current_occupancy_rate | float | Porcentaje de ocupación actual (0-100) |
| total_spaces | integer | Total de espacios en las sedes seleccionadas |
| occupied_spaces | integer | Espacios ocupados actualmente |
| today_revenue | decimal | Ingresos del día en la moneda del tenant |
| today_transactions | integer | Número de transacciones del día |
| avg_occupancy_24h | float | Ocupación promedio últimas 24h |
| last_updated | timestamp | Timestamp de la última actualización |

### Datos históricos (para gráficos)

| Campo | Tipo | Descripción |
|-------|------|-------------|
| timestamp | datetime | Punto en el tiempo |
| occupancy_rate | float | Porcentaje de ocupación |
| revenue | decimal | Ingresos en ese período |
| transaction_count | integer | Número de transacciones |

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| No hay datos para la sede seleccionada | Mostrar "Sin datos" en el widget; no mostrar error. |
| Sitio está temporalmente desconectado (IoT down) | Mostrar el último dato conocido con un indicador de "Datos puede no estar actualizado". |
| Rango de fechas muy amplio (> 1 año) | Rechazar; máximo 365 días. |
| Dashboard solicitado desde plan no Enterprise | Retornar 403 con mensaje de upgrade. |
| Servicio de analytics no disponible | Retornar error 503; el frontend muestra "Dashboard temporalmente no disponible". |

---

## Criterios de Aceptación

1. El dashboard carga en < 3 segundos con datos del día actual.
2. Los widgets se actualizan automáticamente cada 60 segundos sin recargar la página.
3. El usuario puede filtrar por sede, zona, rango de fechas y granularidad.
4. Los datos históricos tienen resolución de 1 hora para rangos hasta 30 días, y 1 día para rangos mayores.
5. El acceso al dashboard está restringido a tenants Enterprise+ y roles con permisos de BI.
6. Todos los valores monetarios se muestran en la moneda configurada del tenant.
7. Las tendencias muestran comparison vs el período anterior (ej: "12% más que ayer").

## Endpoints API Involucrados

- `GET /api/v1/analytics/realtime-summary` — Datos en tiempo real para widgets
- `GET /api/v1/analytics/occupancy-history` — Datos históricos de ocupación
- `GET /api/v1/analytics/revenue-history` — Datos históricos de ingresos
- `GET /api/v1/analytics/transaction-history` — Datos históricos de transacciones
- `GET /dashboard/bi` — Página del dashboard (renderizado server-side o SPA)

---

## Métricas de Éxito

- Tiempo de carga inicial del dashboard: < 3 segundos (p95)
- Actualización en tiempo real: cada 60 segundos
- Disponibilidad: 99.9%
- Los datos de ocupación son максимум 2 minutos old (latencia del snapshot).

---

## Notas

- Los datos en tiempo real se sirven desde una tabla de snapshots (`occupancy_snapshots`) que se actualiza cada minuto por un job en background.
- No se debe hacer query a los dispositivos IoT directamente para el dashboard — toda la información pasa por la base de datos.
- Para charting library se puede usar Chart.js o Apache ECharts (ambas soportan streaming updates).
- El dashboard es una SPA (Single Page Application) que consume los endpoints REST del analytics-service.
- Los umbrales de "alerta" (ej: ocupación > 90%) pueden mostrarse con color coding (verde/amarillo/rojo).
