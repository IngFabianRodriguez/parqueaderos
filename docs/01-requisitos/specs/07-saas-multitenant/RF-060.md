# RF-060 — Email de Bienvenida al Crear la Cuenta

## Metadata

- **Módulo**: SaaS Multi-Tenant — Onboarding de Nuevo Tenant
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: notification-service, tenant-service, auth-service

---

## User Story

**Como** nuevo usuario de ParkCore **quiero** recibir un email de bienvenida al crear mi cuenta **para** saber qué hacer a continuación y tener una buena primera impresión de la plataforma.

---

## Objetivo

El sistema debe enviar un email transaccional de bienvenida al usuario founder inmediatamente después de que su cuenta tenant se cree exitosamente (RF-029). El email debe incluir instrucciones claras de próximo paso (verificar email, comenzar onboarding) y un link directo al wizard de onboarding.

---

## Actores

- **Actor principal**: Sistema (envía el email)
- **Actor secundario**: Nuevo usuario (recibe y lee el email)

---

## Precondiciones

- El tenant se creó exitosamente con un email válido.
- El email del founder está en la tabla `users` con `role = tenant_admin` y `status = pending_verification`.

---

## Postcondiciones

### En caso de éxito

- El email de bienvenida se envía al email del founder.
- El email contiene: instrucciones, link al onboarding, link de verificación de email.
- Se genera el evento `email_sent` con template `welcome`.

### En caso de falla

- Si el email no se puede entregar (hard bounce), se marca el usuario como `email_bounced`.
- Si el email falla por causa del provider, se reintenta hasta 3 veces con backoff.

---

## Flujo Funcional

1. El tenant se crea exitosamente (RF-029).
2. El `auth-service` detecta el evento `tenant_created`.
3. El servicio consulta el `tenant_branding` para usar el logo del tenant (si ya tiene branding configurado; si no, usa el default ParkCore).
4. El servicio prepara el email de bienvenida con template `welcome`:
   - **Asunto**: "¡Bienvenido a ParkCore, {nombre}! Completa tu configuración en 5 minutos."
   - **Saludo**: "Hola {nombre},"
   - **Cuerpo**:
     - Bienvenida y descripción breve de ParkCore.
     - Instrucciones de próximo paso: "Verifica tu email para comenzar."
     - Botón primario: "Verificar mi email" (link de verificación, RF-061).
     - Sección "Qué viene después": resumen del onboarding (1-2 líneas).
     - Si el plan seleccionado durante signup tiene features destacadas, mencionarlas.
   - **Footer**: datos del tenant si están disponibles.
5. El email se envía con:
   - `from_address`: `{nombre_tenant} <bienvenida@parkcore.co>` (o custom domain si existe, RF-049).
   - Tracking de apertura y clicks.
6. Se genera evento `welcome_email_sent` en `events_saas`.

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| tenant_id | UUID | Identificador del tenant | Sí |
| user_id | UUID | ID del usuario founder | Sí |
| email | string | Email del usuario | Sí |
| nombre | string | Nombre del usuario | Sí |
| tenant_name | string | Nombre de la empresa del tenant | Sí |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| email_id | string | ID del email en el provider |
| sent_at | datetime | Timestamp de envío |
| template | string | `welcome` |
| opened | boolean | Si el email fue abierto (async, se actualiza después) |
| clicked | boolean | Si se hizo clic en algún link |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| El email es hard bounce | Marcar `users.email_bounced = true`; no enviar más emails a esta dirección. |
| Soft bounce temporal | Reintentar con backoff exponencial (1h, 4h, 12h). |
| El usuario ya verificó su email | El botón de verificar no aparece; el email incluye link al dashboard. |
| El template de email no existe | Usar template genérico de ParkCore; loguear error. |
| El branding del tenant no está disponible | Usar branding default de ParkCore. |

---

## Criterios de Aceptación

1. El email de bienvenida se envía dentro de los 60 segundos posteriores a la creación del tenant.
2. El email incluye el nombre del usuario y el nombre de su empresa.
3. El botón "Verificar mi email" apunta al link de verificación (RF-061).
4. El email es responsive ( legible en móvil).
5. El tracking de apertura y clicks está habilitado.
6. Si el email bouncea, se marca el usuario entsprechend y se alerta al equipo.
7. El evento `welcome_email_sent` se registra en `events_saas`.

---

## Endpoints API Involucrados

- Este es un proceso batch triggered por `tenant_created`; no hay endpoint directo para el usuario.

---

## Métricas de Éxito

- Emails de bienvenida entregados: > 95%.
- Tasa de apertura del email de bienvenida: benchmark 40-50%.
- Tiempo entre envío y primera apertura: < 1 hora (benchmark).
- Bounce rate < 2%.

---

## Notas

- El email de bienvenida es el primer touchpoint con el nuevo usuario; debe ser cálido y profesional.
- Si el plan seleccionado tiene features distintivas, se mencionan para generar expectativa positiva.
- El email incluye la dirección de soporte por si el usuario tiene dudas.
- El `from_address` usa el branding del tenant si está disponible (RF-048).
