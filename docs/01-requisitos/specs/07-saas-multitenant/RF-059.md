# RF-059 — Wizard de Onboarding para Nuevo Tenant

## Metadata

- **Módulo**: SaaS Multi-Tenant — Onboarding de Nuevo Tenant
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: onboarding-service, tenant-service, notification-service

---

## User Story

**Como** nuevo usuario de ParkCore **quiero** que el sistema me guíe paso a paso en la configuración inicial de mi cuenta **para** tener mi parqueadero funcionando rápidamente sin necesidad de leer documentación.

---

## Objetivo

El sistema debe presentar un wizard de onboarding que guíe al nuevo tenant a través de los pasos esenciales para configurar su cuenta: datos de la empresa, creación de la primera sede, definición de zonas y espacios, configuración de la primera talanquera, e invitación del equipo. Al completar cada paso, el sistema guarda la configuración y avanza al siguiente.

---

## Actores

- **Actor principal**: Nuevo usuario (founder del tenant)
- **Actor secundario**: Sistema (guía y guarda progreso)

---

## Precondiciones

- El tenant se creó exitosamente (RF-029).
- El usuario fundador tiene email verificado (RF-061).
- El tenant está en estado `trial`.

---

## Postcondiciones

### En caso de éxito

- La configuración inicial del tenant está completa.
- Al menos una sede está creada con al menos una zona y espacios.
- El usuario fundador tiene acceso completo al dashboard.
- El progreso del onboarding se marca como `completed`.

### En caso de abandono

- El progreso se guarda automáticamente en cada paso.
- Si el usuario cierra la sesión y vuelve, el wizard se reanuda desde donde quedó.
- El email de recordatorio se envía si el onboarding no se completa en 48 horas.

---

## Flujo del Wizard (6 Pasos)

### Paso 1: Datos de la Empresa

1. El wizard muestra: "Configuremos tu empresa".
2. Campos obligatorios:
   - **Nombre de la empresa**: texto (ya viene pre-poblado si se ingresaron en RF-029).
   - **NIT**: texto (validación de formato colombiano).
   - **Dirección**: texto.
   - **Ciudad**: dropdown de ciudades colombianas.
   - **Teléfono de contacto**: texto.
   - **Sector**: dropdown (parqueadero, centro comercial, hotel, etc.).
3. El usuario hace clic en \"Siguiente\".
4. El sistema guarda en `tenants`; avanza al paso 2.

### Paso 2: Tu Primera Sede

1. El wizard muestra: \"Ahora configuremos tu primera sede\".
2. Campos:
   - **Nombre de la sede**: texto (ej: \"Sede Centro\").
   - **Dirección**: texto.
   - **Ciudad**: dropdown.
   - **Horario de operación**: hora inicio y hora fin (para reportes).
   - **Modo de operación**: `iot` (con dispositivos) o `manual` (sin dispositivos, ver RF-083).
3. Si se selecciona `iot`, se muestra sección adicional: \"Configurar dispositivos (opcional, se puede hacer después)\".
4. El usuario hace clic en \"Siguiente\".
5. El sistema crea la sede en `sedes`; avanza al paso 3.

### Paso 3: Zonas y Espacios

1. El wizard muestra: \"Define las zonas y espacios de tu sede\".
2. Sistema ofrece templates predefinidos (RF-062):
   - **Template Básico**: 1 zona, 20 espacios (10 covered, 10 uncovered).
   - **Template Profesional**: 2 zonas (Cubierta, Descubierta), 40 espacios.
   - **Template Completo**: 3 zonas (VIP, Cubierta, Descubierta), 80 espacios.
   - **Empezar desde cero**: permite definir manualmente.
3. El usuario selecciona un template o crea manualmente.
4. Para cada zona: nombre, tipo de espacio (cubierto/descubierto/VIP/moto), cantidad de espacios.
5. El usuario hace clic en \"Siguiente\".
6. El sistema crea zonas y espacios en `zonas` y `espacios`; avanza al paso 4.

### Paso 4: Configuración de Talanqueras (solo modo IoT)

1. Si se eligió modo `iot` en paso 2:
   a. Sistema muestra: \"Configura tus talanqueras\" (RF-086).
   b. El usuario puede añadir talanqueras con: nombre, tipo (entrada/salida), IP del dispositivo.
   c. Puede saltar este paso y hacerlo después desde el admin.
2. Si se eligió modo `manual`: se salta este paso.
3. El usuario hace clic en \"Siguiente\"; avanza al paso 5.

### Paso 5: Tarifa Inicial

1. El wizard muestra: \"Define tu primera tarifa\".
2. Sistema sugiere la tarifa más común basada en el sector:
   - **Por hora**: $X/hora (sugerido según sector).
   - **Tarifa nocturna**: opcional.
   - **Tarifa mensual**: opcional.
   - **Fracción mínima**: 15 minutos.
3. El usuario ajusta o acepta los valores sugeridos.
4. El usuario hace clic en \"Siguiente\".
5. El sistema crea la tarifa en `tarifas`; avanza al paso 6.

### Paso 6: Invitar al Equipo

1. El wizard muestra: \"Último paso: invita a tu equipo\".
2. El usuario puede:
   - Invitar hasta 3 usuarios (email, rol: manager/operador).
   - Omitir este paso y hacerlo después.
3. Si invita: se envía email de invitación (RF-033).
4. El usuario hace clic en \"Finalizar\".
5. Sistema marca `onboarding_completed_at = NOW()`; `onboarding_status = completed`.
6. Sistema redirige al dashboard del tenant.

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| empresa_nombre | string | Nombre de la empresa | Sí |
| nit | string | NIT de la empresa | Sí |
| direccion | string | Dirección de la empresa | Sí |
| ciudad | string | Ciudad de la empresa | Sí |
| telefono | string | Teléfono de contacto | No |
| sector | string | Sector/industria | No |
| sede_nombre | string | Nombre de la primera sede | Sí |
| sede_direccion | string | Dirección de la sede | Sí |
| modo_operacion | enum | `iot` o `manual` | Sí |
| zonas | array | Array de zonas con espacios | Sí |
| talanqueras | array | Array de talanqueras (si modo iot) | No |
| tarifa | object | Tarifa inicial (tipo, precio, fracción) | Sí |
| usuarios_invitar | array | Emails de usuarios a invitar | No |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| tenant_id | UUID | Tenant configurado |
| onboarding_status | enum | `in_progress`, `completed` |
| completed_at | datetime | Timestamp de finalización |
| first_sede_id | UUID | ID de la primera sede creada |
| setup_progress | percentage | Porcentaje de wizard completado |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Usuario cierra el navegador durante el wizard | El progreso se guarda automáticamente; al volver, se reanuda. |
| Error de validación en un paso | Se muestra mensaje inline en el campo; no se avanza hasta corregir. |
| El usuario abandona el wizard (no completa) | Se envía email recordatorio en 48h; el estado del tenant queda `onboarding_pending`. |
| NIT duplicado en el sistema | Rechazar con mensaje \"Este NIT ya está registrado. Contacta a soporte.\" |
| Ciudad no encontrada en el dropdown | Permitir escritura libre; guardar como especificado. |

---

## Criterios de Aceptación

1. El wizard guía al usuario a través de los 6 pasos en orden, sin poder saltar pasos (excepto talanqueras si modo manual).
2. El progreso se guarda automáticamente al completar cada paso.
3. Si el usuario regresa a la plataforma, el wizard se reanuda desde el último paso completado.
4. La primera sede, zonas y espacios se crean correctamente en la base de datos.
5. El usuario fundador puede invitar hasta 3 usuarios al final del wizard.
6. Al completar el wizard, el tenant tiene al menos: 1 sede, 1 zona, 1 espacio, 1 tarifa.
7. Si el onboarding no se completa en 48 horas, se envía email recordatorio.

## Dependencias

**RFs relacionados**:
- RF-029 — Creación del tenant ( inicio del onboarding)
- RF-061 — Verificación de email con OTP (prerrequisito del wizard)
- RF-062 — Asistente de configuración de primera sede (templates de zonas)
- RF-033 — Múltiples usuarios por tenant ( invitación al equipo)
- RF-083 — Modo de operación por sede ( paso 4 del wizard)
- RF-086 — Cambio de modo de operación ( puede volver a esto después)

**Tablas de base de datos afectadas**:
- `tenants` — columnas de empresa, `onboarding_status`, `onboarding_completed_at`
- `sedes` — primera sede creada
- `zonas` — zonas de la sede
- `espacios` — espacios de cada zona
- `tarifas` — primera tarifa
- `onboarding_progress` — tracking de pasos completados

---

## Endpoints API Involucrados

- `POST /api/v1/onboarding/step/{step}` — Guardar progreso de un paso
- `GET /api/v1/onboarding/status` — Consultar estado del onboarding
- `POST /api/v1/tenants/{tenant_id}/sedes` — Crear primera sede
- `POST /api/v1/tenants/{tenant_id}/zonas` — Crear zonas

---

## Métricas de Éxito

- Onboarding completion rate: meta > 70% de nuevos tenants completan el wizard.
- Tiempo promedio de completion: < 15 minutos.
- Drop-off por paso: identificar qué paso tiene más abandonos.
- Primera transacción feita en menos de 1 día post-onboarding: meta 50%.

---

## Notas

- El wizard es bloqueante: el usuario no puede acceder al dashboard completo hasta completarlo (pero puede ver su progreso).
- Los templates de zonas/espacios (RF-062) usan datos predefinidos para acelerar el setup.
- El wizard es mobile-friendly para usuarios que prefieren configurar desde el celular.
- Los usuarios invitados durante el onboarding reciben un email con un enlace especial que omite el onboarding y los lleva directo al dashboard.
