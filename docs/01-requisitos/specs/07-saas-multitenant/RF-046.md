# RF-046 — Configuración de Branding por Tenant (Logo, Colores)

## Metadata

- **Módulo**: SaaS Multi-Tenant — Branding y Personalización
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: branding-service, tenant-service, asset-service

---

## User Story

**Como** administrador de un tenant **quiero** configurar mi logo, color primario y color secundario **para** que la plataforma refleje la identidad visual de mi empresa en todos los puntos de contacto.

---

## Objetivo

El sistema debe permitir que cada tenant personalice su identidad visual (logo, color primario, color secundario) desde su panel de administración, almacenando estos assets de forma segura y sirviéndolos dinámicamente en todas las interfaces donde aparezca el branding del tenant.

---

## Actores

- **Actor principal**: Tenant admin (configura el branding)
- **Actor secundario**: Sistema (sirve assets), Operadores y usuarios del tenant (consumen el branding)

---

## Precondiciones

- El tenant tiene estado `trial` o `active`.
- El tenant admin tiene sesión activa.
- Los assets (logo) cumplen con los requisitos de formato y tamaño.

---

## Postcondiciones

### En caso de éxito

- Los datos de branding del tenant se guardan y se aplican inmediatamente en el dashboard.
- Los assets subidos (logo) se almacenan en un bucket de assets con una URL pública.
- El email transaccional muestra el logo y los colores del tenant.
- El dashboard del operador muestra los colores configurados del tenant.

### En caso de falla

- Si el archivo de logo no cumple formato (PNG, JPG, SVG), se muestra error de validación.
- Si el archivo excede el tamaño máximo (2MB), se rechaza.
- Los valores por defecto (logo ParkCore, colores #2563EB y #1E40AF) se usan si no hay configuración.

---

## Flujo Funcional

1. Tenant admin accede a Settings > Branding.
2. Sistema muestra formulario con: campo de logo (upload), selector de color primario (hex), selector de color secundario (hex), preview en tiempo real.
3. Tenant admin sube imagen de logo:
   a. Sistema valida formato (PNG, JPG, SVG), tamaño (máx 2MB), dimensiones (mín 128x128px, máx 512x512px).
   b. Si pasa validación: se sube a storage (S3 o equivalente) con path `branding/{tenant_id}/logo.{ext}`.
   c. Se genera URL pública del asset.
4. Tenant admin selecciona color primario y secundario usando el picker.
5. Sistema guarda en la tabla `tenant_branding`: `tenant_id`, `logo_url`, `primary_color`, `secondary_color`, `updated_at`.
6. Sistema genera preview en tiempo real: el panel derecho muestra cómo se verá el dashboard con los colores elegidos.
7. Tenant admin hace clic en \"Guardar cambios\".
8. Sistema actualiza `tenant_branding` y genera evento `branding_updated`.
9. El branding se aplica instantáneamente en todas las interfaces del tenant.

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| logo_file | file | Imagen PNG/JPG/SVG (máx 2MB, 128x128 a 512x512px) | No (default: logo ParkCore) |
| primary_color | string | Color primario en hex (#RRGGBB) | Sí (default: #2563EB) |
| secondary_color | string | Color secundario en hex (#RRGGBB) | Sí (default: #1E40AF) |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| logo_url | string | URL pública del logo almacenado |
| primary_color | string | Color primario configurado |
| secondary_color | string | Color secundario configurado |
| preview_url | string | URL de preview renderizado del branding |
| updated_at | datetime | Timestamp de última modificación |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Logo no es PNG/JPG/SVG | Rechazar con error \"Formato no soportado. Usa PNG, JPG o SVG.\" |
| Logo excede 2MB | Rechazar con error \"El archivo excede el límite de 2MB.\" |
| Logo es muy pequeño (<128px) | Rechazar con error \"La imagen debe ser mínimo 128x128 píxeles.\" |
| Color hex inválido | Rechazar con error \"Color inválido. Usa formato hex (#RRGGBB).\" |
| Tenant no configura branding | Se usan valores por defecto; no hay error. |
| Storage no disponible al subir | Retornar 503; el usuario puede reintentar. |

---

## Criterios de Aceptación

1. El tenant admin puede subir un logo y seleccionar colores desde Settings > Branding.
2. El logo se almacena en storage con URL pública accesible.
3. Los colores y logo se guardan en la tabla `tenant_branding` y se sirven en todas las pantallas del tenant.
4. El preview en tiempo real muestra cómo se verá el dashboard antes de guardar.
5. Los cambios se aplican inmediatamente después de guardar (< 2 segundos).
6. Si el tenant no tiene branding configurado, se usan los valores por defecto de ParkCore.
7. El logo se muestra en el email transaccional (RF-048).
8. Los colores se aplican en la barra superior, botones primarios y acentos del dashboard del tenant.

## Endpoints API Involucrados

- `GET /api/v1/tenants/{tenant_id}/branding` — Obtener branding actual
- `PUT /api/v1/tenants/{tenant_id}/branding` — Actualizar branding (logo + colores)
- `POST /api/v1/assets/upload` — Subir logo al storage

---

## Métricas de Éxito

- Branding configurado por tenant: objetivo 100% de tenants activos tienen al menos colores configurados.
- Tiempo de aplicación post-guardado < 2 segundos.
- Precisión del preview: coincide 100% con el resultado final.

---

## Notas

- Los logos se redimensionan automáticamente a 256x256px para storage optimized.
- Los colores se validan contra una lista de colores accesibles (contraste WCAG AA mínimo).
- El campo `logo_url` puede ser null si el tenant no ha subido logo; el sistema usa el default.
- El branding se aplica en el subdomain del tenant (`{slug}.parkcore.co`) y cualquier custom domain (RF-049).
