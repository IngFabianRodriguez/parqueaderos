# RF-062 — Asistente de Configuración de Primera Sede con Templates Predefinidos

## Metadata

- **Módulo**: SaaS Multi-Tenant — Onboarding de Nuevo Tenant
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: onboarding-service, tenant-service, sede-service

---

## User Story

**Como** nuevo usuario de ParkCore **quiero** elegir un template predefinido para configurar mi primera sede **para** tener mi parqueadero listo en minutos sin necesidad de definir cada espacio manualmente.

---

## Objetivo

El sistema debe ofrecer un conjunto de templates predefinidos de configuración de sede (con zonas, tipos de espacio y cantidades recomendadas) para que el usuario pueda seleccionar uno durante el onboarding y aplicarlo con un solo clic, en lugar de crear todo desde cero.

---

## Actores

- **Actor principal**: Nuevo usuario (selecciona y aplica el template)
- **Actor secundario**: Sistema (aplica el template seleccionado)

---

## Precondiciones

- El tenant está en el paso 3 del wizard de onboarding (RF-059).
- La primera sede ya fue creada (paso 2 del wizard).

---

## Postcondiciones

### En caso de éxito

- Las zonas y espacios del template se crean en la sede seleccionada.
- El usuario puede editar o eliminar las zonas/espacios después si lo desea.
- El progreso del onboarding avanza al siguiente paso.

---

## Flujo Funcional

1. En el paso 3 del wizard (Zonas y Espacios), el sistema muestra los templates disponibles.
2. El usuario ve una representación visual de cada template con preview de la distribución.
3. El usuario selecciona un template y hace clic en \"Aplicar\".
4. El sistema aplica el template:
   - Crea las zonas definidas en el template.
   - Para cada zona, crea los espacios con el tipo y cantidad especificados.
   - Genera los espacios con numeración automática (Zona A: A-01, A-02...; Zona B: B-01...).
5. El usuario ve la sede configurada y puede:
   - Editar nombres de zonas o espacios.
   - Añadir más zonas o espacios.
   - Eliminar zonas o espacios preconfigurados.
   - O aceptar tal como está y avanzar.
6. El sistema guarda la configuración en `zonas` y `espacios`.

---

## Templates Predefinidos

### Template 1: Básico

**Descripción**: Para parqueadero pequeño, un nivel, hasta 20 espacios.

| Zona | Tipo | Espacios |
|------|------|----------|
| General | Descubierto | 20 |

### Template 2: Profesional

**Descripción**: Para parqueadero mediano, dos zonas diferenciadas, hasta 40 espacios.

| Zona | Tipo | Espacios |
|------|------|----------|
| Cubierta | Cubierto | 20 |
| Exterior | Descubierto | 20 |

### Template 3: Corporativo

**Descripción**: Para parqueadero empresarial, tres zonas incluyendo VIP, hasta 80 espacios.

| Zona | Tipo | Espacios |
|------|------|----------|
| VIP | Cubierto | 10 |
| Cubierta | Cubierto | 30 |
| Exterior | Descubierto | 40 |

### Template 4: Aeropuerto / Centro Comercial

**Descripción**: Alta capacidad, múltiples zonas, hasta 200 espacios.

| Zona | Tipo | Espacios |
|------|------|----------|
| Principal | Descubierto | 80 |
| Cubierta | Cubierto | 60 |
| VIP | Cubierto | 20 |
| Moto | Descubierto | 40 |

### Template 5: Empezar desde cero

**Descripción**: No aplica ningún template; el usuario define todo manualmente.

---

## Visualización de Templates

Cada template se muestra como:
- **Nombre**: Título del template.
- **Descripción**: Una línea descriptiva.
- **Preview visual**: Diagrama de la distribución de zonas y espacios.
- **Indicadores**: Número de zonas, número total de espacios.

El usuario puede hacer hover sobre el template para ver más detalle de cada zona.

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| sede_id | UUID | Identificador de la sede | Sí |
| template_id | enum | Identificador del template seleccionado | Sí |
| zonas_personalizadas | array | overrides opcionales sobre el template | No |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| zonas_creadas | array | Lista de zonas creadas con sus espacios |
| total_espacios | integer | Cantidad total de espacios creados |
| template_usado | string | Nombre del template aplicado |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Template produce más espacios que el límite del plan | Mostrar warning: \"Este template excede el límite de tu plan. Puedes eliminar espacios después.\" |
| El usuario modifica el template después de aplicar | Los cambios se guardan normalmente; ya no está vinculado al template. |
| Template no disponible para el plan | No se muestra; solo los templates apropiados según el plan. |
| El usuario selecciona \"desde cero\" | El sistema permite crear zonas manualmente sin template. |

---

## Criterios de Aceptación

1. El sistema ofrece al menos 4 templates predefinidos más la opción de empezar desde cero.
2. Cada template tiene una representación visual clara de la distribución de zonas.
3. Al aplicar un template, las zonas y espacios se crean correctamente en la base de datos.
4. El usuario puede modificar cualquier zona o espacio después de aplicar el template.
5. Los templates disponibles se filtran según el plan del tenant (templates grandes no aparecen para Starter).
6. El progreso del onboarding se marca correctamente después de aplicar el template.

## Dependencias

**RFs relacionados**:
- RF-059 — Wizard de onboarding ( paso donde se seleccionan los templates)
- RF-037 — Selección de plan ( para filtrar templates según el plan)

**Tablas de base de datos afectadas**:
- `zonas` — zonas creadas por el template
- `espacios` — espacios creados por el template
- `onboarding_progress` — tracking de pasos completados

---

## Endpoints API Involucrados

- `GET /api/v1/templates/sede` — Listar templates de sede disponibles
- `POST /api/v1/sedes/{sede_id}/apply-template` — Aplicar un template a la sede

---

## Métricas de Éxito

- Templates utilizados: el más popular debe ser usado por > 40% de los nuevos tenants.
- Tiempo promedio de configuración de zonas con template vs manual: reducción > 50%.
- Satisfacción del usuario con los templates (encuesta post-onboarding): > 4/5.

---

## Notas

- Los templates son definiciones de configuración, no código; se almacenan como JSON en la tabla `sede_templates`.
- El usuario puede agregar templates adicionales personalizados después del onboarding (guardados como \"Mis templates\").
- Los templates no son rígidos: después de aplicar, todo es editable.
