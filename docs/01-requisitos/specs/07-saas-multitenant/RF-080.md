# RF-080 — Gestión de Límites de Gasto por Vehículo dentro de una Flota

## Metadata

- **Módulo**: SaaS Multi-Tenant — Módulo B2B Flotas (Enterprise+)
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: fleet-service, billing-service

---

## User Story

**Como** administrador de una empresa B2B con flota de vehículos **quiero** configurar límites de gasto individuales para cada vehículo de mi flota **para** controlar el gasto y evitar que un vehículo en particular exceda el presupuesto asignado.

---

## Objetivo

El sistema debe permitir que el admin de la empresa B2B configure límites de gasto mensuales por vehículo. Cuando un vehículo alcance su límite, el sistema debe bloquear el acceso al estacionamiento (la talanquera no abre) o aplicar una tarifa diferenciada. Los límites pueden ser uniformes para toda la flota o específicos por vehículo.

---

## Actores

- **Actor principal**: Admin de la empresa B2B (`company_admin`)
- **Actor secundario**: Administrador del tenant (`tenant_admin`) — puede overrided límites

---

## Precondiciones

- La empresa B2B está registrada y tiene un plan empresarial asignado (RF-079).
- La empresa tiene al menos un vehículo registrado en su flota.

---

## Postcondiciones

### En caso de éxito

- El límite se guarda en `vehicle_spending_limits`.
- Cuando el vehículo excede el límite, se aplica la acción configurada (bloqueo o tarifa especial).
- El consumo mensual se reinicia al comenzar cada ciclo de facturación.

### En caso de falla

- Si el vehículo no pertenece a la empresa del admin: error de permisos.
- Si el límite es inválido: error de validación.

---

## Flujo Funcional

### Configurar límite por vehículo

1. El `company_admin` accede a `Fleet → Vehicles → [Vehicle] → Spending Limit`.
2. Define el límite mensual para ese vehículo:
   - `monthly_limit`: Monto en la moneda del tenant.
   - `action_when_exceeded`: `block` (bloquear) o `surcharge` (aplicar tarifa diferenciada).
   - `surcharge_rate`: Porcentaje adicional a aplicar como tarifa (si eligió `surcharge`).
3. Guarda la configuración.
4. El sistema actualiza `vehicle_spending_limits`.

### Aplicación del límite (en check-in/check-out)

1. El vehículo con matrícula `ABC-123` llega a la talanquera.
2. El sistema identifica el vehículo y busca su `vehicle_spending_limit`.
3. Si el vehículo no tiene límite individual, se usa el límite por defecto del plan de la empresa (RF-079).
4. El sistema consulta `vehicles_usage` para obtener el gasto acumulado en el mes actual.
5. Si `gasto_mes_actual + estimado_del_estacionamiento > limite`:
   - Si `action_when_exceeded = block`: la talanquera no abre; se muestra mensaje "Límite de gasto excedido. Contacte al administrador de su empresa."
   - Si `action_when_exceeded = surcharge`: se permite el acceso pero se marca el vehículo para aplicar la tarifa diferencial en el checkout.
6. Al final del mes, el consumo se resetea para el nuevo ciclo de facturación.

### Configurar límite uniforme para toda la flota

1. El `company_admin` puede ir a `Fleet → Settings → Default Spending Limit`.
2. Define un límite default que aplica a todos los vehículos de la flota que no tengan límite individual configurado.

---

## Datos de Entrada

### Límite por vehículo

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| vehicle_id | uuid | ID del vehículo | Sí |
| monthly_limit | decimal | Límite de gasto mensual | Sí |
| action_when_exceeded | enum | `block` o `surcharge` | Sí |
| surcharge_rate | float | Porcentaje de recarga (si action = surcharge) | No (requerido si action = surcharge) |

### Límite por defecto de flota

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| default_monthly_limit | decimal | Límite default para toda la flota | Sí |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| vehicle_id | uuid | ID del vehículo |
| monthly_limit | decimal | Límite configurado |
| action_when_exceeded | string | Acción al exceder |
| current_spend | decimal | Gasto acumulado en el mes actual |
| remaining_budget | decimal | Presupuesto restante |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Vehículo no tiene límite configurado | Usar el límite default del plan de la empresa. |
| Límite = 0 | Significa "sin límite" (ilimitado). |
| Gasto exactamente en el límite | Permitir el acceso (no bloquear hasta exceder). |
| Admin de empresa B2B intenta setear límite mayor al del plan | El sistema permite pero muestra warning de que excede el límite del plan. |
| El vehículo se transfiere a otra empresa | Los límites se eliminan; la nueva empresa configura sus propios límites. |
| Ciclo de facturación se reinicia | El consumo acumulado se resetea a 0; los límites se mantienen. |

---

## Criterios de Aceptación

1. El `company_admin` puede configurar límites individuales por vehículo.
2. Si un vehículo no tiene límite individual, se aplica el límite default de la flota.
3. Cuando el gasto mensual alcanza el límite, la acción configurada (bloqueo o recarga) se ejecuta sin intervención manual.
4. El bloqueo de talanquera por límite de gasto se ejecuta en < 500ms para no causar delay en la entrada.
5. El admin de la empresa B2B y el `tenant_admin` pueden ver el gasto acumulado y剩余 presupuesto de cada vehículo.
6. Los límites pueden modificarse en cualquier momento; los cambios aplican inmediatamente.
7. El límite de la empresa (RF-079) actúa como tope máximo; no se puede configurar un límite vehicular mayor al límite de la empresa.

## Dependencias

**RFs relacionados**:

- RF-005, RF-006 — Check-in/check-out (donde se aplica el límite)

**Tablas de base de datos afectadas**:
- `vehicle_spending_limits` — límites por vehículo
- `vehicles` — información del vehículo y su empresa
- `vehicles_usage` — tracking de gasto mensual por vehículo

---

## Endpoints API Involucrados

- `GET /api/v1/companies/{company_id}/vehicles/{vehicle_id}/spending-limit` — Ver límite de un vehículo
- `PUT /api/v1/companies/{company_id}/vehicles/{vehicle_id}/spending-limit` — Configurar límite
- `GET /api/v1/companies/{company_id}/fleet/spending-summary` — Resumen de gasto de toda la flota
- `PUT /api/v1/companies/{company_id}/default-spending-limit` — Configurar límite default de la flota

---

## Métricas de Éxito

- Tiempo de verificación de límite en check-in: < 100ms (cache en Redis).
- Bloqueo efectivo: 100% de los vehículos que exceden el límite y tienen `action=block` son bloqueados.
- Cero vehículos con límite configurado que no se les aplica la regla.

---

## Notas

- El gasto acumulado mensual se calcula como la suma de los `amount` de las transacciones del vehículo en el mes calendario actual.
- Si la empresa tiene `billing_cycle = prepaid`, el límite funciona como crédito disponible; cuando se agota, se bloquea.
- Para latency crítica en la talanquera, la verificación de límite debe usar Redis con el valor actual del gasto y el límite configurado.
- Se puede implementar un "soft limit" que envía una notificación al admin de la empresa cuando el vehículo alcanza el 80% del límite, antes de bloquear.
