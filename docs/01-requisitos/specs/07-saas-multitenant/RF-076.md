# RF-076 — Benchmarking de Métricas entre Sedes

## Metadata

- **Módulo**: SaaS Multi-Tenant — BI Avanzado (solo Enterprise+)
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: analytics-service, bi-service

---

## User Story

**Como** administrador de un tenant Enterprise con múltiples sedes **quiero** comparar las métricas de operación entre sedes **para** identificar cuáles están tendo mejor desempeño y cuáles necesitan atención.

---

## Objetivo

El sistema debe permitir al usuario visualizar y comparar métricas clave (ocupación, ingresos, transacciones, tiempo promedio de permanencia) entre las distintas sedes de su organización. La comparación puede ser en un momento dado (snapshot) o a lo largo del tiempo (tendencia). Los datos se нормализованы para permitir comparación equitativa entre sedes de diferentes tamaños.

---

## Actores

- **Actor principal**: Administrador del tenant (`tenant_admin`)
- **Actor secundario**: Usuario con rol `bi_viewer`

---

## Precondiciones

- El tenant tiene plan Enterprise o superior.
- El tenant tiene al menos 2 sedes configuradas.
- El actor tiene permisos de visualización de BI.

---

## Postcondiciones

### En caso de éxito

- El sistema presenta una tabla y gráficos comparativos entre sedes.
- El usuario puede ordenar las sedes por cualquier métrica.
- El usuario puede ver el ranking de cada sede.

### En caso de falla

- Si hay menos de 2 sedes: mostrar mensaje indicando que se necesitan al menos 2 sedes para benchmarking.

---

## Flujo Funcional

1. El usuario accede a `GET /dashboard/bi/benchmarking` o desde el dashboard principal hace clic en "Comparar sedes".
2. El sistema verifica que haya al menos 2 sedes en el tenant.
3. El sistema consulta las métricas para cada sede en el período seleccionado:
   - Para cada sede: ocupación promedio, ingresos totales, transacciones, ticket promedio, tiempo promedio de permanencia.
   - Los datos se normalizan: para ocupación se usa porcentaje; para ingresos se usa la moneda del tenant.
4. Se presenta una tabla comparativa con las métricas.
5. Se presentan gráficos de comparison:
   - Barras agrupadas para ingresos por sede.
   - Líneas de tendencia para ocupación sobre el tiempo.
   - Tabla de ranking ordenada por métrica seleccionada.

### Métricas comparadas

| Métrica | Descripción | Normalización |
|---------|-------------|---------------|
| Ocupación (%) | Porcentaje promedio de espacios ocupados | Sin normalización (porcentaje) |
| Ingresos ($) | Total de ingresos en el período | Por sede (no normalizar, pero el usuario puede filtrar por tamaño) |
| Transacciones (#) | Número total de transacciones | Conteo absoluto |
| Ticket promedio ($) | Ingresos / Transacciones | Cálculo derivado |
| Estancia promedio (min) | Tiempo medio de permanencia | Minutos |
| Rotación (vehículos/espacio) | Transacciones / Número de espacios | Por espacio (permite comparar sedes de distinto tamaño) |
| Ingresos por espacio ($) | Ingresos / Número de espacios | Normalizado por tamaño de sede |

### Cálculo de benchmarking

- **Ranking**: Las sedes se ordenan por la métrica seleccionada, de mejor a peor.
- **vs. Promedio**: Se calcula cuánto está una sede por encima o debajo del promedio de todas las sedes.
- **vs. Meta**: Si el tenant tiene metas definidas por sede, se muestra el % de cumplimiento.

## Datos de Salida

### Tabla comparativa

| Campo | Tipo | Descripción |
|-------|------|-------------|
| site_id | uuid | ID de la sede |
| site_name | string | Nombre de la sede |
| occupancy_rate | float | Ocupación promedio (%) |
| revenue | decimal | Ingresos totales |
| transactions | integer | Total de transacciones |
| avg_ticket | decimal | Ticket promedio |
| avg_stay_minutes | float | Estancia promedio en minutos |
| rotation_per_space | float | Rotación por espacio |
| revenue_per_space | decimal | Ingresos por espacio |
| rank | integer | Posición en el ranking de la métrica seleccionada |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Menos de 2 sedes | Mensaje: "Se requieren al menos 2 sedes para comparar." |
| Sede sin transacciones en el período | Mostrar 0 en todas las métricas; no mostrar error. |
| Metas no configuradas | Mostrar "N/A" en la columna de cumplimiento vs meta. |
| Todos los valores de una métrica son 0 | Evitar división por cero en cálculos derivados (ticket promedio, etc.). |
| Benchmarking con filtro de fecha muy granular | Si hay pocos datos, mostrar mensaje "Datos insuficientes para el período seleccionado." |

---

## Criterios de Aceptación

1. El benchmarking muestra mínimo las métricas: ocupación, ingresos, transacciones, ticket promedio, estancia promedio.
2. El usuario puede ordenar la tabla por cualquier columna.
3. Las sedes se muestran con su nombre legible, no solo el ID.
4. Los gráficos comparativos permiten visualizar tendencias de cada sede en el tiempo.
5. Los datos de ocupación se normalizan por el total de espacios, permitiendo comparar sedes de distintos tamaños.
6. Se indica claramente cuál es la sede con mejor y peor desempeño en cada métrica.
7. El acceso está restringido a tenants Enterprise+ con rol adecuado.

---

## Métricas de Éxito

- Tiempo de generación del reporte de benchmarking: < 5 segundos para hasta 50 sedes.
- Los datos se refrescan cada 5 minutos (no en tiempo real como RF-075).
- Cobertura: 100% de las sedes del tenant en el reporte.

---

## Notas

- La métrica de "Rotación por espacio" es especialmente útil para comparar sedes de diferentes tamaños: una sede con 100 espacios y 500 transacciones tiene rotación 5; una con 50 espacios y 300 transacciones tiene rotación 6.
- Se puede implementar un "heat map" donde cada celda de la matriz sitio x métrica tenga un color según el percentil.
- El benchmarking permite identificar no solo las mejores/pires sedes, sino también anomalías (ej: una sede con ocupación muy alta pero ingresos bajos podría indicar un problema de pricing).
