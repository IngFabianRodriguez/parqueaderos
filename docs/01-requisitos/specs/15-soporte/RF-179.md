# RF-179 — Solicitud de Feedback Post-Transacción

## Metadata

- **Módulo**: 15-soporte
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: feedback-service, notificacion-service, soporte-service

---

## User Story

**Como** cliente **quiero** calificar mi experiencia después de cada transacción de parqueo **para** que el operador sepa cómo fue mi experiencia y pueda mejorar el servicio.

---

## Objetivo

Recolectar feedback estructurado del cliente al finalizar cada transacción de parqueo (salida del vehículo), mediante una calificación de 1-5 estrellas y un campo de comentario opcional. El sistema debe enviar la solicitud automáticamente después de que se confirma el pago y la salida.

---

## Actores

- **Actor principal**: Cliente de la app
- **Actores secundarios**: Sistema de notificaciones

---

## Precondiciones

- El cliente debe haber completado al menos una transacción de parqueo (entrada y salida)
- El pago debe estar confirmado (RF-016 / RF-017)
- El cliente debe tener notificaciones push habilitadas (o proporcionar email alternativo)

---

## Postcondiciones

### En caso de éxito

- El cliente recibe solicitud de feedback vía push notification o email
- El feedback queda registrado con la calificación, comentario y fecha
- El NPS del cliente se recalcula si es cliente recurrente
- La calificación queda visible para el operador de la sede

### En caso de falla

- El sistema reintenta enviar la solicitud hasta 3 veces en 24 horas
- Si no se completa, el feedback queda como "pendiente" para colección manual

---

## Flujo Funcional

1. Cuando se confirma la salida del vehículo y el pago está completado, sistema registra `transaccion_completada_at`
2. Sistema espera 30 segundos para permitir que el cliente abandone la zona de influencia de la talanquera
3. Sistema envía push notification al cliente con título "Cómo estuvo tu parqueo?" y cuerpo "Cuéntanos tu experiencia en [nombre_sede] para ayudar-nos a mejorar"
4. Notificación incluye botón "Calificar ahora" que abre la interfaz de feedback
5. Si el cliente no tiene push habilitadas, se envía email con link directo a la encuesta
6. Cliente ve pantalla de feedback con:
   - Nombre de la sede y fecha/hora de la transacción
   - Calificación: 5 estrellas (excelente) a 1 estrella (muy malo)
   - Campo opcional: "Cuéntanos qué te gustó o qué podemos mejorar" (texto 500 máx)
   - Botón "Enviar"
7. Cliente selecciona estrellas y opcionalmente escribe comentario
8. Cliente presiona "Enviar"
9. Sistema almacena feedback y calcula/actualiza NPS del cliente
10. Sistema muestra mensaje de agradecimiento y cierra la interfaz
11. Si el cliente no responde en 24 horas, sistema reintenta hasta 2 veces más (días 2 y 3)
12. Si después de 72 horas no hay respuesta, el feedback queda como "no respondido" para métricas

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Cliente no tiene app instalada (solo email) | Enviar email con link a encuesta web responsive |
| Cliente elimina cuenta después de recibir solicitud | Marcar feedback como "anonimizado" sin asociar a cliente |
| Cliente califica con 1 estrella | Sistema marca como "feedback negativo" y alerta al operador de sede |
| Comentario incluye palabras prohibidas | Filtrar contenido inapropiado; guardar asteriscos en lugar de palabras |
| Push notification no se entrega | Caer a email como canal secundario; reintentar hasta 3 veces |
| Cliente intenta calificar dos veces la misma transacción | Mostrar "Ya has calificado esta transacción" con su calificación anterior |

---

## Criterios de Aceptación

1. Sistema envía solicitud de feedback a todo cliente que completa una transacción exitosa
2. Cliente puede completar la encuesta en menos de 30 segundos
3. Feedback queda almacenado y visible para el operador de la sede dentro de 1 minuto de ser enviado
4. Calificación de 1-2 estrellas genera alerta inmediata al operador
5. Sistema reintenta hasta 3 veces si el cliente no responde en las primeras 72 horas
6. El NPS del cliente se recalcula con cada nuevo feedback recibido
7. Reporte de calificaciones está disponible en dashboard del operador (ver RF-180)

## Métricas de Éxito

- Tasa de respuesta de encuestas: > 40%
- Calificación promedio de satisfacción: > 3.5 estrellas
- Tiempo promedio para completar encuesta: < 30 segundos
- Porcentaje de transacciones con feedback negativo (1-2 estrellas) alertadas: 100%

---

## Notas

- La notificación push debe incluir el logo del tenant para personalización (RF-048)
- Para clientes con múltiples transacciones, solo solicitar feedback 1 vez por semana máximo (evitar spam)
- Las calificaciones se agregan por sede para calcular el "rating" de la sede (mostrado en app cliente)
- Si el cliente tiene un ticket de soporte abierto para esa transacción, no se envía encuesta hasta que el ticket se resuelva
- El sistema debe agrupar calificaciones por semana para no abrumar al operador con alertas individuales
- Implementar "smart timing": si el cliente históricamente califica entre 8am-10am, enviar en ese horario
