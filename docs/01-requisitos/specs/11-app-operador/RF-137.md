# RF-137 — Estado en Tiempo Real de Talánqueras en App

## Metadata

- **Módulo**: 11 — App Operador
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: gate-service, iot-service, operator-app, websocket-service

---

## User Story

**Como** operador de parqueadero **quiero** ver el estado en tiempo real de cada talanquera (abierta/cerrada/error/offline) **para** saber cuáles están operativas y cuáles requieren atención.

---

## Objetivo

La app móvil del operador muestra una vista en tiempo real del estado de todas las talánqueras de la sede. Cada talanquera se muestra con un indicador visual claro de su estado, actualizado instantáneamente ante cualquier cambio mediante WebSocket (o polling como fallback). El operador puede tocar una talanquera para ver más detalle.

---

## Actores

- **Actor principal**: Operador de parqueadero (`operator_role`)
- **Actor secundario**: Supervisor de sede

---

## Precondiciones

- El operador tiene sesión activa en la app.
- Existe al menos una talanquera configurada en la sede.

---

## Postcondiciones

### En caso de éxito

- Todas las talánqueras visibles con su estado actual.
- Los cambios de estado se reflejan automáticamente.

### En caso de falla

- Si no hay conexión, se muestran los últimos estados conocidos en caché.
- Si una talanquera no reporta en > 60s, se marca como offline.

---

## Flujo Funcional

1. El operador abre la pantalla de talánqueras desde el menú o dashboard.
2. El sistema muestra la lista de talánqueras con su estado actual.
3. Cada talanquera se muestra como un card con:
   - Nombre/identificador de la talanquera.
   - Tipo: ENTRADA o SALIDA.
   - Estado visual: color e ícono.
   - Última actividad (hace X minutos).
4. El sistema mantiene conexión WebSocket para actualizar en tiempo real.
5. Ante cualquier cambio de estado, la UI se actualiza instantáneamente.
6. El operador puede tocar una talanquera para ver el detalle (historial de últimas 10 acciones).

## Estados de Talanquera y Representación Visual

| Estado | Color | Ícono | Descripción |
|--------|-------|-------|-------------|
| ABIERTA | Verde | 🟢 puerta abierta | Talanquera en posición abierta |
| CERRADA | Rojo | 🔴 puerta cerrada | Talanquera en posición cerrada |
| EN_MOVIMIENTO | Amarillo | 🟡 en transición | Talanquera moviéndose entre posiciones |
| ERROR | Rojo + ⚠️ | ⚠️ ícono warning | Sensor de error o falla mecánica |
| OFFLINE | Gris | ⚫ gris | Sin comunicación con el IoT |
| MANTENIMIENTO | Naranja | 🔧 | En mantenimiento temporal |

## Datos de Salida (Lista)

| Campo | Tipo | Descripción |
|-------|------|-------------|
| talanquera_id | UUID | Identificador único |
| nombre | String | Nombre asignado (ej: "Entrada Principal") |
| tipo | Enum | ENTRADA, SALIDA |
| estado | Enum | ABIERTA, CERRADA, EN_MOVIMIENTO, ERROR, OFFLINE, MANTENIMIENTO |
| ultimo_update | DateTime | Timestamp del último reporte |
| ultima_accion | Enum | ABRIR, CERRAR, ERROR, TIMEOUT |
| tiempo_estado | String | Cuánto tiempo lleva en este estado (ej: "2 min") |
| sede_id | UUID | ID de la sede |

---

## Datos de Salida (Detalle por Talanquera)

| Campo | Tipo | Descripción |
|-------|------|-------------|
| talanquera_id | UUID | Identificador |
| nombre | String | Nombre completo |
| tipo | Enum | ENTRADA, SALIDA |
| estado | Enum | Estado actual |
| modelo | String | Modelo del dispositivo IoT |
| firmware_version | String | Versión del firmware |
| ultima_conexion | DateTime | Último heartbeat recibido |
| historial_acciones | Array | Últimas 10 acciones con timestamp |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Sin conexión WebSocket | Caer a polling cada 10 segundos; mostrar banner "Modo offline" |
| Talanquera offline (>60s sin heartbeat) | Tarjeta en gris con etiqueta "Offline" |
| Talanquera en error | Mostrar descripción del error si está disponible |
| Talanquera en movimiento | Mostrar animación de transición |
| Sin talánqueras configuradas | Mensaje "No hay talánqueras en esta sede" |
| Error al cargar lista | Retry automático; mensaje de error tras 3 intentos |

---

## Criterios de Aceptación

1. Todos los estados de talánqueras son visibles en una sola pantalla sin scroll horizontal.
2. Los cambios de estado se reflejan en la UI en menos de 2 segundos.
3. El operador puede distinguir el tipo de cada talanquera (entrada/salida).
4. Las talánqueras offline se muestran claramente diferenciadas.
5. El sistema usa WebSocket como canal principal; polling como fallback.
6. Cada talanquera es táctil y abre su historial de últimas acciones.
7. Se muestra el tiempo transcurrido desde la última actividad.
8. La pantalla no se queda en blanco ante fallos de red; siempre hay contenido visible.

## Endpoints API Involucrados

- `GET /api/v1/operator/gates` — Lista de talánqueras con estados
- `GET /api/v1/operator/gates/{id}` — Detalle de una talanquera
- `GET /api/v1/operator/gates/{id}/history` — Último historial de acciones
- `WS /ws/v1/operator/gates` — Canal WebSocket para actualizaciones en tiempo real

---

## Métricas de Éxito

- La lista de talánqueras carga en < 2 segundos.
- Las actualizaciones en tiempo real llegan al cliente en < 1 segundo.
- El fallback a polling ocurre en < 5 segundos tras perder WebSocket.
- 0 estados congelados por más de 60 segundos (sin actualización).

---

## Notas

- La pantalla de talánqueras debe ser accesible como widget rápido desde el dashboard.
- Se debe poder filtrar por tipo (solo entradas, solo salidas).
- Si una talanquera está en ERROR, debe aparecer primero en la lista (priorización).
- El historial de acciones debe incluir: quién la operó, hora, acción, resultado.
