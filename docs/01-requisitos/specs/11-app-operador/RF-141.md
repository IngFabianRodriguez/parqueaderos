# RF-141 — Bloqueo y Desbloqueo de Vehículo desde App

## Metadata

- **Módulo**: 11 — App Operador
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: vehicle-service, customer-service, operator-app, notification-service

---

## User Story

**Como** operador de parqueadero **quiero** bloquear o desbloquear un vehículo desde la app con confirmación de contraseña **para** restringir el acceso de vehículos con mora o infracciones y permitir el ingreso cuando el cliente regularice su situación.

---

## Objetivo

El operador puede bloquear un vehículo (impidiendo su salida hasta que se regularice el pago) o desbloquearlo (habilitando nuevamente su salida) desde la app móvil. Ambas acciones requieren confirmación mediante ingreso de contraseña del operador para evitar uso no autorizado.

---

## Actores

- **Actor principal**: Operador de parqueadero (`operator_role`)
- **Actor secundario**: Supervisor de sede

---

## Precondiciones

- El operador tiene sesión activa en la app.
- El operador tiene permisos para bloquear/desbloquear vehículos.
- El vehículo existe en el sistema.

---

## Postcondiciones

### En caso de éxito — Bloqueo

- El vehículo pasa a estado BLOQUEADO.
- Si el vehículo tiene un ticket abierto, se muestra una alerta.
- El sistema registra el bloqueo con operador, timestamp y motivo.
- El cliente recibe notificación push del bloqueo (si tiene la app instalada).

### En caso de éxito — Desbloqueo

- El vehículo pasa a estado ACTIVO.
- Si había un ticket abierto, se muestra para que el operador gestione el pago.
- El sistema registra el desbloqueo con operador, timestamp y motivo.
- El cliente recibe notificación push del desbloqueo.

### En caso de falla

- Si la contraseña es incorrecta, no se ejecuta la acción.
- Si el vehículo ya está en el estado solicitado, se muestra mensaje.

---

## Flujo Funcional — Bloquear Vehículo

1. El operador busca el vehículo por placa (RF-140) o desde un ticket abierto.
2. El operador toca "Bloquear Vehículo".
3. Se muestra un modal de confirmación con:
   - Datos del vehículo: placa, marca, color.
   - Estado actual.
   - Campo para seleccionar motivo del bloqueo:
     - Mora de pago (automóvil con deuda).
     - Vehículo no autorizado.
     - Solicitud del propietario.
     - Otro.
4. El operador ingresa su contraseña para confirmar.
5. El operador toca "Bloquear".
6. El sistema valida la contraseña.
7. El sistema actualiza el estado del vehículo a BLOQUEADO.
8. Se registra en el log: operador, timestamp, motivo, vehículo.
9. Se muestra confirmación: "Vehículo ABC-123 bloqueado".
10. Se notifica al cliente si es posible.

---

## Flujo Funcional — Desbloquear Vehículo

1. El operador busca el vehículo por placa (RF-140) o desde el panel de bloqueos.
2. El operador toca "Desbloquear Vehículo".
3. Se muestra un modal de confirmación con:
   - Datos del vehículo: placa, marca, color.
   - Estado actual: BLOQUEADO.
   - Campo para ingresar motivo del desbloqueo (opcional).
4. El operador ingresa su contraseña para confirmar.
5. El operador toca "Desbloquear".
6. El sistema valida la contraseña.
7. El sistema actualiza el estado del vehículo a ACTIVO.
8. Se registra en el log: operador, timestamp, motivo, vehículo.
9. Se muestra confirmación: "Vehículo ABC-123 desbloqueado".
10. Si había un ticket abierto, se muestra el detalle para gestionar el pago.

---

## Validaciones

| Campo | Regla |
|-------|-------|
| contraseña | Requerida; debe coincidir con la del operador |
| placa | Requerida; debe existir |
| estado actual | El sistema valida antes de ejecutar |
| vehículo ya bloqueado | Al bloquear, error "Este vehículo ya está bloqueado" |
| vehículo ya activo | Al desbloquear, error "Este vehículo no está bloqueado" |
| sesión activa | La contraseña se verifica contra la sesión activa |

---

## Motivos de Bloqueo

| Código | Descripción |
|--------|-------------|
| MORA_PAGO | Vehículo con deuda pendiente de pago |
| NO_AUTORIZADO | Vehículo no autorizado para ingresar |
| SOLICITUD_PROPIETARIO | Solicitud directa del propietario |
| SANCION | Sanción por infracciones reiteradas |
| MANTENIMIENTO | Bloqueo temporal por mantenimiento |
| OTRO | Otro motivo (requiere observación) |

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Contraseña incorrecta | Se muestra "Contraseña incorrecta" y se permite reintentar (máx 3) |
| Vehículo con ticket abierto al bloquear | Se muestra warning pero se permite bloquear igual |
| Vehículo bloqueado con ticket abierto | Al desbloquear se muestra el ticket para gestionar |
| Operador sin permisos de bloqueo | Error "No tienes permisos para esta acción" |
| Contraseña incorrecta 3 veces | Se bloquea la sesión del operador por 5 minutos |
| Vehículo no existe | Error "No se encontró el vehículo" |
| Cliente sin notificación configurada | Se registra pero no se envía push |

---

## Criterios de Aceptación

1. El operador puede bloquear un vehículo en menos de 10 segundos.
2. El operador puede desbloquear un vehículo en menos de 10 segundos.
3. La contraseña del operador es requerida para ambas acciones.
4. El vehículo queda inmediatamente en el nuevo estado tras la confirmación.
5. El log registra: operador, timestamp, motivo, vehículo, estado anterior/nuevo.
6. La acción de desbloqueo solo se permite en vehículos con estado BLOQUEADO.
7. Si el vehículo tiene un ticket abierto al bloquear, se muestra el detalle.
8. El cliente recibe push notification del bloqueo/desbloqueo si es posible.
9. No se permite bloquear un vehículo que ya está bloqueado.

## Endpoints API Involucrados

- `POST /api/v1/operator/vehicles/{id}/block` — Bloquear vehículo
- `POST /api/v1/operator/vehicles/{id}/unblock` — Desbloquear vehículo
- `GET /api/v1/operator/vehicles/{id}/status` — Estado actual del vehículo
- `GET /api/v1/operator/vehicles/blocked` — Lista de vehículos bloqueados

---

## Métricas de Éxito

- Acción de bloqueo/desbloqueo completa en < 5 segundos.
- El vehículo refleja el nuevo estado en < 1 segundo en el sistema.
- Log registrado en < 1 segundo.
- Notificación enviada (si aplica) en < 10 segundos.

---

## Notas

- El operador debe poder bloquear desde el perfil del cliente (RF-140) o desde un ticket abierto.
- Se recomienda mostrar una animación visual al bloquear/desbloquear para confirmación clara.
- El log de bloqueos debe ser inmutable; no se permite editar ni eliminar registros.
- Si el vehículo tiene múltiples bloqueos, se muestra el historial completo.
