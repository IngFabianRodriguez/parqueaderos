# RF-134 — Registro Manual de Salida y Cálculo de Valor a Pagar

## Metadata

- **Módulo**: 11 — App Operador
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: ticket-service, pricing-service, payment-service, operator-app

---

## User Story

**Como** operador de parqueadero **quiero** registrar la salida de un vehículo manualmente y ver el cálculo del valor a pagar **para** cobrar correctamente al conductor en operaciones donde el vehículo no tiene ticket automático o hay discrepancia.

---

## Objetivo

El operador debe poder buscar un ticket abierto por placa o número de ticket, y al registrar la salida, el sistema debe calcular automáticamente el valor a pagar mostrando un resumen detallado: tiempo de permanencia, tarifas aplicadas, descuentos, y total a pagar. El operador puede confirmar la salida o cancelar si hay algún problema.

---

## Actores

- **Actor principal**: Operador de parqueadero (`operator_role`)
- **Actor secundario**: Supervisor de sede, Cajero

---

## Precondiciones

- El operador tiene sesión activa en la app.
- Existe al menos un ticket ABIERTO en el sistema.

---

## Postcondiciones

### En caso de éxito

- El ticket pasa a estado CERRADO con fecha de salida registrada.
- Se genera la obligación de pago (monto calculado).
- El espacio queda disponible.

### En caso de falla

- Si se cancela, no se modifica ningún registro.
- Si el ticket no existe, se muestra error.

---

## Flujo Funcional

1. El operador toca "Registrar Salida" en el dashboard o menú.
2. El operador busca el ticket por:
   - Escanear código QR del ticket.
   - Escribir la placa.
   - Escribir el número de ticket.
3. El sistema muestra los datos del ticket encontrado: placa, hora de entrada, zona, espacio.
4. El operador confirma que es el vehículo correcto y toca "Calcular".
5. El sistema calcula el monto a pagar:
   - Tiempo transcurrido (entrada → ahora).
   - Tarifa de la zona según tabla de tarifas.
   - Aplicar fracciones (ej: cada 15 minutos).
   - Descuentos o promociones si aplica.
   - Monto mínimo si aplica.
6. Se muestra el resumen al operador:
   - Placa y número de ticket.
   - Hora de entrada y hora actual.
   - Duración: X horas Y minutos.
   - Tarifa aplicada: $X/hora.
   - Subtotal, descuentos, total.
   - **Total a pagar: $MMM**
7. El operador muestra este total al conductor.
8. El operador toca "Confirmar Salida" para proceder al pago (RF-135), o "Cancelar" para abortar.

## Validaciones

| fecha_hora_entrada | DateTime | Timestamp de entrada |

| duracion_minutos | Integer | Tiempo total en minutos |

| subtotal | Decimal | Monto antes de descuentos |
| descuentos | Decimal | Monto descontado |

| estado_ticket | Enum | ABIERTO (pendiente de pago) |
| puede_cerrar | Boolean | Si el ticket puede ser cerrado |

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Ticket no encontrado | Muestra "No se encontró ticket para esa placa/número" |
| Ticket ya cerrado | Muestra "Este ticket ya fue cerrado el [fecha]" |
| Ticket bloqueado | Muestra "Este ticket está bloqueado. Contacte al supervisor" |
| Tarifa no definida para zona | Muestra error "Zona sin tarifa configurada" y no permite calcular |
| Duración 0 minutos | Muestra monto mínimo (tarifa mínima) |
| Cálculo con decimales | Redondear a 2 decimales; mostrar siempre formato moneda |
| Operador cancela | Se cierra la pantalla; no se crea ningún registro |
| Ticket con múltiples días | Mostrar desglose: X días + Y horas + Z minutos |

---

## Criterios de Aceptación

1. La búsqueda de ticket devuelve resultados en < 2 segundos.
2. El cálculo de tarifas se realiza correctamente según la tabla de tarifas de la sede.
3. El resumen muestra claramente el total a pagar al conductor.
4. El ticket queda en estado ABIERTO pero con obligación de pago hasta que se cierre.
5. El espacio se marca como disponible solo al confirmar el cierre.
6. Si el operador cancela, ningún registro se modifica.
7. El registro de salida manual tiene marca "MANUAL" en el log.
8. El operador puede editar la fecha/hora de salida antes de confirmar si hubo un error.

## Endpoints API Involucrados

- `GET /api/v1/operator/tickets/search?placa={placa}` — Buscar ticket por placa
- `GET /api/v1/operator/tickets/{id}` — Detalle del ticket
- `POST /api/v1/operator/tickets/{id}/calculate-exit` — Calcular monto a pagar
- `POST /api/v1/operator/tickets/{id}/exit` — Confirmar salida

---

## Métricas de Éxito

- Búsqueda de ticket en < 2 segundos.
- Cálculo de tarifa en < 500ms.
- Presentación del resumen en < 1 segundo tras el cálculo.
- 0 errores de cálculo de tarifas en operaciones normales.

---

## Notas

- El operador NO cierra el ticket en este paso; aquí solo se registra la salida y se calcula el monto. El cierre definitivo ocurre tras el registro del pago (RF-135).
- Si el vehículo tiene un bloqueo activo, se debe mostrar alerta al operador antes de permitir la salida.
- La pantalla de resumen debe permitir al operador tocar "Imprimir ticket" para generar una versión impresa del resumen de salida.
