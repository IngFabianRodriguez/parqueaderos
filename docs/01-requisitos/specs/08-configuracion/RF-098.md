# RF-098 — Configuración de Layout de la App Móvil del Operador

## Metadata

- **Módulo**: Configuración — App Móvil y UX
- **Prioridad**: Media
- **Tipo**: Funcional
- **Servicios afectados**: mobile-app-service, operator-app, tenant-service

---

## User Story

**Como** administrador de tenant **quiero** poder cambiar el orden de las secciones en la app móvil del operador **para** priorizar las funciones más usadas y adaptar la interfaz a los flujos de trabajo específicos de cada estacionamiento.

---

## Objetivo

El sistema debe permitir a los administradores de tenant personalizar el orden de las secciones principales de la app móvil del operador. Las secciones configurables incluyen: Dashboard/Resumen, Registro de Entrada, Registro de Salida, Búsqueda de Vehículos, Historial, y Configuración. Cada operador puede ver el orden configurado por su administrador de tenant al iniciar sesión.

---

## Actores

- **Actor principal**: Administrador de tenant (configura el layout)
- **Actor secundario**: Operador (ve y usa el layout configurado)
- **Actor terciario**: Sistema (aplica la configuración en la app)

---

## Precondiciones

- El tenant tiene la app móvil de operador habilitada.

---

## Postcondiciones

### En caso de éxito

- La configuración se guarda y activa.
- Los operadores ven el nuevo orden de secciones al abrir la app.
- Se genera un evento `app_layout_config_updated`.

### En caso de falla

- La configuración no se guarda y se retorna error.
- Los operadores continúan con el layout anterior.

| Section ID | Nombre | Descripción | ¿Default Visible? |
|------------|--------|-------------|-------------------|

## Estructura de Configuración de Layout

| Campo | Tipo | Descripción |
|-------|------|-------------|
| layout_id | uuid | Identificador único |
| tenant_id | uuid | Tenant al que pertenece |
| profile_name | string | Nombre del perfil (ej: "Operador Rápido", "Supervisor") |
| description | string | Descripción del perfil |
| sections | array | Lista ordenada de secciones |
| show_icons | boolean | Si se muestran iconos en las secciones |
| default_section | string | Sección que se muestra al abrir la app |
| compact_mode | boolean | Modo compacto (más información en pantalla) |
| active | boolean | Si es el perfil activo |
| created_at | datetime | Fecha de creación |
| updated_at | datetime | Última modificación |

### Estructura de cada Sección

| Campo | Tipo | Descripción |
|-------|------|-------------|
| section_id | string | Identificador de la sección |
| order | integer | Orden de aparición (1 = primero) |
| visible | boolean | Si la sección se muestra |
| badge_count | boolean | Si muestra contador/badge (ej: vehículos bloqueados) |
| shortcut_enabled | boolean | Si se puede acceder desde shortcut rápido |

## Flujo Funcional

### Configuración del layout por el administrador

1. El administrador accede a **Configuración → App Operador → Layout de Secciones**.
2. Ve la lista de secciones disponibles con su orden actual.
3. Arrastra y suelta para reordenar las secciones.
4. Alternativamente, usa los botones de subir/bajar para cambiar el orden.
5. Puede ocultar secciones desmarcando "Visible".
6. Define cuál es la sección por defecto al abrir la app.
7. Guarda la configuración.
8. Se persiste en `app_layout_configs`.
9. Se genera evento `app_layout_config_updated`.

### Aplicación del layout en la app del operador

1. El operador abre la app.
2. La app consulta `GET /api/v1/operators/me/app-layout`.
3. El backend retorna el layout activo para el tenant + operador.
4. La app renderiza las secciones en el orden configurado.
5. Los módulos no visibles según el layout se ocultan o deshabilitan.
6. Si el operador toca una sección oculta, se muestra "Acceso no disponible".

### Cambio de layout en tiempo real

1. El administrador cambia el layout.
2. Se genera evento `app_layout_config_updated`.
3. Los operadores que tienen la app abierta reciben push notification.
4. La app actualiza el layout en el siguiente refresh o al reabrir.

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Todas las secciones ocultas | Error de validación; al menos una debe estar visible |
| Sección por defecto oculta | Se usa la primera sección visible como fallback |
| El layout se elimina mientras operadores lo usan | Los operadores ven el layout por defecto del sistema |
| Cambio de layout mientras operador está en una sección oculta | La sección sigue accesible hasta que se cierre la app |
| Un operador tiene un layout personalizado (por rol) | El layout por tenant tiene menor prioridad que el por rol |
| La app no puede descargar el layout (sin conexión) | Se usa el layout cacheado localmente |
| Nueva sección agregada por el sistema | Aparece al final del layout; no altera el orden existente |

---

## Criterios de Aceptación

1. Un administrador puede reordenar las secciones de la app del operador.
2. Las secciones disponibles son: Dashboard, Entrada, Salida, Búsqueda, Historial, Bloqueados, Reportes, Configuración, Ayuda.
3. Se pueden crear múltiples perfiles de layout (ej: "Rápido", "Supervisor").
4. Se puede asignar un perfil como activo para el tenant.
5. Los operadores ven el nuevo orden al reabrir la app o tras notificación de actualización.
6. Se puede ocultar secciones completas para operadores que no las necesitan.
7. La sección por defecto se puede cambiar.
8. El layout se aplica por tenant, pero puede overridarse por rol de operador.
9. Se soportan cambios de layout en tiempo real sin necesidad de actualizar la app.

## Endpoints API Involucrados

- `GET /api/v1/tenants/{tenant_id}/app-layouts` — Listar layouts
- `POST /api/v1/tenants/{tenant_id}/app-layouts` — Crear layout
- `PUT /api/v1/tenants/{tenant_id}/app-layouts/{id}` — Actualizar layout
- `DELETE /api/v1/tenants/{tenant_id}/app-layouts/{id}` — Eliminar layout
- `POST /api/v1/tenants/{tenant_id}/app-layouts/{id}/activate` — Activar layout
- `GET /api/v1/operators/me/app-layout` — Obtener layout para el operador actual

---

## Métricas de Éxito

- Número de layouts custom por tenant: promedio 1-2.
- Tiempo promedio de reordenamiento de secciones: < 30 segundos.
- Porcentaje de uso de cada sección (para identificar cuáles mover arriba).
- Tiempo hasta que operadores ven el nuevo layout tras cambio: < 5 minutos.

---

## Notas

- Los perfiles de layout permiten tener diferentes configuraciones para diferentes roles de operador.
- El "Modo compacto" reduce el espacio visual de cada sección para mostrar más información.
- Los badges/contadores en las secciones (ej: "3 vehículos bloqueados") requieren sync en background.
- Se recomienda que la sección de Entrada sea la primera para operadores de alta rotación.
- Los cambios de layout son atómicos: o se aplica todo el nuevo orden o se mantiene el anterior.
