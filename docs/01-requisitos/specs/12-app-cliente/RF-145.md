# RF-145 — Gestión de Vehículos del Cliente

## Metadata

- **Módulo**: App Móvil del Cliente
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: vehiculo_service, app-cliente-backend

---

## User Story

**Como** cliente **quiero** agregar, editar y eliminar vehículos de mi cuenta **para** poder usar diferentes Autos al hacer parqueo y mantener mi información actualizada.

---

## Objetivo

El sistema debe permitir al cliente gestionar su lista de vehículos desde la app. Cada vehículo tiene placa, marca, modelo, color y tipo de vehículo. El cliente puede tener hasta un máximo de 5 vehículos por cuenta.

---

## Actores

- **Actor principal**: Cliente (usuario final autenticado)
- **Actores secundarios**: Servicio de validación de placas

---

## Precondiciones

- El cliente debe haber iniciado sesión en la app.
- El cliente no debe haber alcanzado el límite máximo de vehículos (5).

---

## Postcondiciones

### En caso de éxito

- El vehículo se crea/actualiza/elimina de la cuenta del cliente.
- El vehículo aparece/no aparece en la lista de selección al hacer un parqueo.
- Los parqueos activos asociados a un vehículo eliminado siguen vigentes hasta su cierre natural.

### En caso de falla

- El vehículo no se modifica y se muestra error al cliente.

---

## Flujo Funcional

### Agregar Vehículo

1. El cliente accede a "Mi Cuenta" → "Mis Vehículos" → "Agregar Vehículo".
2. El sistema presenta un formulario con: placa, marca, modelo, año, color, tipo (carro/moto/camión).
3. El cliente llena los datos y confirma.
4. El sistema valida que la placa no esté ya registrada en la plataforma por otro cliente.
5. El sistema guarda el vehículo asociado a la cuenta del cliente.
6. Se muestra confirmación y el vehículo queda disponible para seleccionarlo en parkings.

### Editar Vehículo

1. El cliente selecciona un vehículo de "Mis Vehículos".
2. El cliente pulsa "Editar".
3. El sistema permite modificar todos los campos excepto la placa (por ser identificador único).
4. El cliente confirma los cambios.
5. El sistema actualiza los datos del vehículo.
6. Se muestra confirmación.

### Eliminar Vehículo

1. El cliente selecciona un vehículo de "Mis Vehículos".
2. El cliente pulsa "Eliminar".
3. El sistema muestra confirmación: "Este vehículo está en 2 parqueos activos. Si lo eliminas, no podrás seleccionarlo para nuevos parqueos hasta que los parqueos se cierren. ¿Continuar?".
4. Si el cliente confirma:
   a. El vehículo se marca como "inactivo" en la base de datos.
   b. No se elimina el registro (para mantener trazabilidad).
   c. El vehículo desaparece de la lista activa del cliente.
5. Si el vehículo tiene parqueo activo, queda visible en "Parqueos Activos" pero no seleccionable para nuevos parqueos.

|-------|------|-------------|-----------|

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| La placa ya existe en otro cliente | Se muestra error: "Esta placa ya está registrada en otra cuenta" |
| Límite de 5 vehículos alcanzado | Se muestra mensaje indicando el límite y opción de eliminar uno existente |
| Placa con formato inválido | Se muestra mensaje indicando el formato esperado (ej: ABC123) |
| Cliente intenta editar la placa | Se muestra error indicando que la placa no es editable |
| Cliente intenta eliminar vehículo con parqueo activo | Se permite con advertencia; el vehículo queda inactivo pero visible en parqueo activo |
| Cliente intenta eliminar el último vehículo | Se permite; la cuenta queda sin vehículos (debe agregar otro para hacer prepagos) |

---

## Criterios de Aceptación

1. El cliente puede agregar hasta 5 vehículos a su cuenta.
2. El cliente puede editar todos los campos de un vehículo excepto la placa.
3. El cliente puede eliminar un vehículo, lo cual lo marca como inactivo.
4. La placa se valida contra formato y disponibilidad antes de guardar.
5. El cliente puede marcar un vehículo como "default" para que aparezca primero en cada parqueo.
6. El vehículo default se pre-selecciona automáticamente al hacer un prepago (RF-143).
7. Al eliminar un vehículo con parqueo activo, el vehículo sigue apareciendo en ese parqueo específico.
8. El sistema impide registrar la misma placa dos veces en la plataforma.

## Dependencias

**RFs relacionados**:

- RF-147 — Historial (muestra los vehículos usados en transacciones pasadas)
- RF-141 — Bloquear/desbloquear vehículo (el operador bloquea por placa)

**Tablas de base de datos afectadas**:
- `vehiculos` — CRUD de vehículos por cliente
- `clientes` — Relación con vehículo default

---

## Endpoints API Involucrados

- `GET /api/v1/cliente/vehiculos` — Lista los vehículos del cliente
- `POST /api/v1/cliente/vehiculos` — Crea un nuevo vehículo
- `PUT /api/v1/cliente/vehiculos/{vehiculo_id}` — Actualiza un vehículo
- `DELETE /api/v1/cliente/vehiculos/{vehiculo_id}` — Elimina (desactiva) un vehículo
- `PUT /api/v1/cliente/vehiculos/{vehiculo_id}/default` — Define el vehículo default

---

## Métricas de Éxito

- Promedio de vehículos por cliente activo > 1.5
- Tasa de intentos de placa duplicada < 1% del total de registros

---

## Notas

- La placa es el identificador principal para la integración con los lectores ANPR en la entrada/salida.
- El vehículo default se usa como sugerencia al cliente al hacer prepago.
- Los vehículos inactivos se mantienen en la BD para trazabilidad histórica de parqueos.
