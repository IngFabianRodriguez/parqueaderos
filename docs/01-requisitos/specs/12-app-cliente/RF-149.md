# RF-149 — Configuración de Notificaciones del Cliente

## Metadata

- **Módulo**: App Móvil del Cliente
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: notificacion_service, app-cliente-backend

---

## User Story

**Como** cliente **quiero** configurar qué notificaciones quiero recibir y por qué canal (push, SMS o email) **para** controlar la comunicación y recibir solo la información que me interesa.

---

## Objetivo

El sistema debe permitir al cliente configurar sus preferencias de notificación desde la app. El cliente puede habilitar o deshabilitar tipos específicos de notificaciones y elegir el canal preferido para cada una.

---

## Actores

- **Actor principal**: Cliente (usuario final autenticado)
- **Actores secundarios**: Servicios de push (FCM/APNS), SMS gateway, servicio de email

---

## Precondiciones

- El cliente debe haber iniciado sesión en la app.
- El cliente debe tener al menos un canal de contacto verificado (email o teléfono).

---

## Postcondiciones

### En caso de éxito

- Las preferencias del cliente se guardan y se aplican a partir del siguiente envío de notificación.

### En caso de falla

- Se mantienen las preferencias anteriores y se muestra error.

---

## Flujo Funcional

1. El cliente accede a "Mi Cuenta" → "Configuración de Notificaciones".
2. El sistema muestra las categorías de notificaciones disponibles y el estado actual de cada una.
3. El cliente puede togglear cada notificación entre habilitada/deshabilitada.
4. Para cada notificación habilitada, el cliente puede seleccionar el canal preferido:
   - Push (requiere tener la app instalada y notificaciones habilitadas en el SO)
   - SMS (requiere número de teléfono verificado)
   - Email (requiere email verificado)
5. El cliente puede seleccionar múltiples canales para una misma notificación.
6. El cliente también ve las opciones de "idioma de notificaciones" y "horario静音" (no molestar).
7. Al guardar, el sistema actualiza las preferencias y muestra confirmación.
8. Si el cliente deshabilita todos los canales de una notificación, esa notificación deja de envíosarse.

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| cliente_id | UUID | ID del cliente | Sí |
| tipo_notificacion | string | Identificador del tipo de notificación | Sí |
| canales | array | Lista de canales habilitados ['push', 'sms', 'email'] | Sí |
| habilitada | boolean | Si la notificación está activa | Sí |
| horario_no_molestar_desde | time | Hora inicio de silencio (null = desactivado) | No |
| horario_no_molestar_hasta | time | Hora fin de silencio | No |
| idioma | string | Código de idioma preferido (ej: 'es', 'en') | No |

---

## Tipos de Notificación Disponibles

| Tipo | Descripción | Canal sugerido |
|------|-------------|----------------|
| recordatorio_vencimiento | Recordatorio de que el parqueo está por vencer | push |
| confirmacion_prepago | Confirmación de un prepago realizado | push, email |
| confirmacion_renovacion | Confirmación de una renovación | push |
| factura_generada | Nueva factura disponible | email |
| promocion | Promociones y descuentos | push, email |
| recordatorio_salida | recordatorio de que el vehículo lleva mucho tiempo | push |
| cambios_cuenta | Cambios en la cuenta o métodos de pago | push, email, sms |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| preferencias_id | UUID | Identificador del registro de preferencias |
| cliente_id | UUID | ID del cliente |
| notificaciones | array | Lista de preferencias por tipo |
| habilitada | boolean | Estado general de notificaciones push |
| canales_configurados | array | Canales disponibles y verificados para el cliente |
| fecha_actualizacion | timestamp | Última modificación |

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Cliente sin email verificado | El canal email aparece deshabilitado y con tooltip "Verifica tu email para habilitar" |
| Cliente sin teléfono verificado | El canal SMS aparece deshabilitado y con tooltip "Verifica tu teléfono para habilitar" |
| Cliente tiene la app desinstalada | El canal push aparece como "No disponible" y se sugiere reinstalar |
| Todas las notificaciones están deshabilitadas | Se muestra un mensaje提醒 "No recibirás alertas de parqueo. ¿Estás seguro?" |
| El horario no molestar se solapa | Se muestra advertencia "El horario se solapa con el día siguiente" |

---

## Criterios de Aceptación

1. El cliente puede habilitar/deshabilitar cada tipo de notificación individualmente.
2. El cliente puede elegir uno o más canales (push/SMS/email) para cada notificación.
3. Los canales no verificados se muestran bloqueados con opción de verificar.
4. El cliente puede configurar un horario de no molestar.
5. Los cambios en preferencias se aplican inmediatamente (no requieren re-login).
6. El cliente recibe solo las notificaciones que ha habilitado.
7. Se muestra el estado actual de cada canal (verificado/no verificado).

## Endpoints API Involucrados

- `GET /api/v1/cliente/config/notificaciones` — Obtiene las preferencias del cliente
- `PUT /api/v1/cliente/config/notificaciones` — Actualiza las preferencias
- `POST /api/v1/cliente/verificar/email` — Envía email de verificación
- `POST /api/v1/cliente/verificar/telefono` — Envía SMS de verificación

---

## Métricas de Éxito

- Tasa de usuarios que personalizan sus notificaciones > 60%
- Tasa de opt-out de notificaciones push < 15%
- Tasa de aperturas de notificaciones > 30%

---

## Notas

- Las notificaciones críticas de seguridad (ej: vehículo bloqueado) se envían por todos los canales disponibles independientemente de las preferencias.
- El horario de silencio (no molestar) se evalúa en el momento del envío; si la notificación es crítica, se envía de todas formas.
- La preferencia de idioma de notificaciones puede diferir del idioma de la app.
