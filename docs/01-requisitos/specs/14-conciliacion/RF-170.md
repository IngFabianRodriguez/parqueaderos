# RF-170 — Marcación Automática de Discrepancias Mayores al 0.5%

## Metadata

- **Módulo**: Conciliación
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: [conciliacion-service, notificacion-service, caja-service]

---

## User Story

**Como** sistema **quiero** marcar automáticamente como "en_discrepancia" toda diferencia mayor al 0.5% entre el efectivo esperado y el efectivo real **para** que el operador y el administrador detecten problemas de caja oportunamente.

---

## Objetivo

El sistema debe aplicar un umbral automático del 0.5% sobre el total de efectivo esperado. Cuando la diferencia calculada exceda este umbral (ya sea por sobrante o faltante), el registro de conciliación debe marcarse con estado "en_discrepancia" y generar una notificación al administrador de sede.

---

## Actores

- Sistema (actor automatizado)
- Operador de caja (secundario, recibe notificaciones)
- Administrador de sede (secundario, recibe alertas de discrepancia)

---

## Precondiciones

- Existe un registro de conciliación calculado (RF-169)
- Se ha calculado la diferencia entre efectivo físico y efectivo esperado

---

## Postcondiciones

### En caso de éxito

- El registro de conciliación tiene estado "en_discrepancia"
- Se genera una alerta/notificación para el administrador de sede
- Se registra en audit log la discrepancia detectada

### En caso de falla

- Si la diferencia es menor o igual al 0.5%, el estado permanece "conciliado"
- Si no se puede generar la notificación, el cálculo de conciliación igual se completa

---

## Flujo Funcional

1. Sistema completa el cálculo de conciliación (RF-169)
2. Sistema obtiene el valor de diferencia y el total esperado
3. Sistema calcula porcentaje = (|diferencia| / total_esperado) * 100
4. Sistema compara porcentaje con umbral = 0.5%
5. Si porcentaje > umbral:
   a. Sistema actualiza estado_conciliacion = "en_discrepancia"
   b. Sistema genera alerta con detalles de la diferencia
   c. Sistema envía notificación push/email al administrador de sede
   d. Sistema registra en audit log: operador, turno, diferencia, timestamp
6. Si porcentaje <= umbral:
   a. Sistema mantiene estado_conciliacion = "conciliado"
   b. No se genera alerta automática

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| diferencia | Decimal | Diferencia calculada (puede ser positiva o negativa) | Sí |
| total_esperado | Decimal | Total de efectivo esperado | Sí |
| turno_id | UUID | Identificador del turno | Sí |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| estado_conciliacion | Enum | "conciliado" / "en_discrepancia" |
| porcentaje_diferencia | Decimal | Porcentaje de la diferencia |
| umbral_aplicado | Decimal | Valor del umbral usado (0.5) |
| alerta_generada | Boolean | Si se generó alerta |
| notificacion_enviada | Boolean | Si se envió notificación al admin |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| total_esperado = 0 (sin transacciones) | No aplicar umbral, marcar como "sin_datos" sin discrepancia |
| diferencia = 0.5% exactamente | Marcar como "conciliado" (umbral es mayor, no mayor o igual) |
| diferencia = 0.51% | Marcar como "en_discrepancia" |
| Diferencia negativa (faltante) | Aplicar mismo umbral, marcar en_discrepancia si |porcentaje| > 0.5% |
| Diferencia positiva (sobrante) | Aplicar mismo umbral, marcar en_discrepancia si porcentaje > 0.5% |
| Administrador no disponible para notificación | Registrar alerta igual, reintentar envío más tarde |

---

## Criterios de Aceptación

1. Toda diferencia mayor al 0.5% en valor absoluto se marca como "en_discrepancia"
2. Diferencias de exactamente 0.5% NO se marcan como discrepancia
3. El cálculo del porcentaje usa el total esperado como base
4. Se genera notificación al admin dentro de los 30 segundos de detectada la discrepancia
5. El registro de auditoría se crea con operador, turno, monto y tipo de diferencia

## Endpoints API Involucrados

- `POST /api/v1/conciliacion/turno/{turno_id}/marcar-discrepancia` — Marcar manualmente como discrepancia
- `GET /api/v1/conciliacion/discrepancias` — Listar todas las discrepancias abiertas

---

## Métricas de Éxito

- Porcentaje de discrepancias detectadas que reciben notificación al admin: 100%
- Tiempo medio de notificación post-detección: < 30 segundos
- Cantidad de falsos positivos (diferencias menores al umbral marcadas incorrectamente): 0

---

## Notas

- El umbral de 0.5% es configurable por el tenant_admin (ver RF-166)
- La notificación al administrador debe incluir: operador, turno, diferencia en monto y porcentaje, link a la conciliación
- Se diferencian discrepancias por sobrante (dinero de más) vs faltante (dinero de menos)
