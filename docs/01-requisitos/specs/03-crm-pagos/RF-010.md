# RF-010 — Registrar clientes naturales y empresas con placas asociadas

## Metadata

- **Módulo**: CRM y Pagos
- **Prioridad**: Alta
- **Tipo**: Funcional
- **Servicios afectados**: crm-service

---

## User Story

**Como** Cliente (registro), Operador (registro asistido) **quiero** registrar un cliente en el sistema con sus datos personales y las placas de sus vehículos asociados **para** permitir que el sistema reconozca al cliente en ingresos futuros y aplique tarifación preferente si corresponde.

---

## Objetivo

Crear y mantener un registro completo del cliente que incluya: tipo (persona natural o empresa), datos de identificación, información de contacto, y lista de placas asociadas. El cliente puede tener múltiples vehículos registrados y un perfil de tarifación especial.

---

## Actores

- Cliente (registro), Operador (registro asistido)
- [Actores secundarios]

---

## Precondiciones

      - El tenant existe y está activo
      - Los datos de identificación no están duplicados en el sistema

---

## Postcondiciones

### En caso de éxito

      - Cliente creado con UUID único
      - Placas asociadas vinculadas al cliente
      - Cliente puede ser reconhecido en entrada/salida por placa

### En caso de falla

      - Duplicado de documento → 409 con datos del cliente existente
      - Tenant no existe → 403

---

## Flujo Funcional

1. Cliente o operador inicia registro desde app móvil o panel admin
1. Sistema recibe POST /api/v1/clientes con datos del cliente
1. Sistema valida que el documento no exista en otro cliente activo
1. Sistema crea registro en tabla cliente
1. Sistema vincula las placas ingresadas creando registros en placa_asociada
1. Si cliente tiene plan prepago, se crea wallet con saldo inicial 0
1. Sistema retorna cliente_id y confirmación

---

## Datos de Entrada

| Campo | Tipo | Descripción | Requerido |
|-------|------|-------------|-----------|
| tipo_cliente | VARCHAR | natural o empresa | Sí |
| documento | VARCHAR | Cédula o NIT | Sí |
| nombre | VARCHAR | Nombre completo o Razón Social | Sí |
| email | VARCHAR | Correo electrónico | Sí |
| telefono | VARCHAR | Teléfono de contacto | Sí |
| placas | ARRAY | Array de placas a asociar | Sí |

---

## Datos de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| cliente_id | UUID | — |
| documento | VARCHAR | — |
| nombre | VARCHAR | — |
| placas_count | INTEGER | — |

---

## Casos de Borde y Manejo de Errores

| Escenario | Comportamiento Esperado |
|-----------|--------------------------|
| Documento ya existe | Retorna 409 con los datos del cliente existente y opción de actualizar |
| Cliente empresa sin NIT verificado | Se crea pero con estado 'pendiente_verificacion' |
| Más de 5 placas por cliente | Se permite hasta 10, más de eso requiere aprobación del admin |

---

## Criterios de Aceptación

- Documento único por tenant (no duplicado)
- Placa validada contra formato colombiano antes de asociar
- Cliente queda inmediatamente activo tras el registro
